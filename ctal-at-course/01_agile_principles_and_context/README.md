# Chapter 1 — Agile Principles and Context for Technical Testers

## Exam Weighting

- **Estimated weight:** ~15% (~6 questions out of 40)
- **K-levels:** K2 (understand), K3 (apply)
- **Focus:** Agile values and principles, Scrum and Kanban, the technical tester's role, agile testing practices

---

## Learning Objectives

By the end of this chapter, you should be able to:

1. Explain the Agile Manifesto values and principles
2. Describe Scrum, Kanban, and XP from a testing perspective
3. Explain the role of the technical tester in agile teams
4. Describe agile testing practices (whole-team approach, continuous feedback)
5. Explain the Definition of Done and Definition of Ready from a testing perspective

---

## Key Concepts

- Agile Manifesto: 4 values, 12 principles
- Scrum: sprints, ceremonies, roles, artifacts
- Kanban: flow, WIP limits, visualization
- XP: TDD, pair programming, refactoring, CI
- Technical tester role in agile teams
- Whole-team approach to quality
- Definition of Done and Definition of Ready
