# 🔧 Chapter 4 — CI/CD, DevOps, and Continuous Testing

## 🎓 Exam Weighting

- **Estimated weight:** ~20% (~8 questions out of 40)
- **K-levels:** K2 (understand), K3 (apply), K4 (analyze)
- **Focus:** CI/CD pipelines, DevOps culture, continuous testing, pipeline design, quality gates

---

## 🎯 Learning Objectives

By the end of this chapter, you should be able to:

1. Explain Continuous Integration (CI) and its testing activities
2. Explain Continuous Delivery and Continuous Deployment (CD)
3. Describe the DevOps culture and the technical tester's role
4. Design a CI/CD pipeline with appropriate testing stages
5. Explain quality gates and their role in the pipeline

---

## ⭐ Key Concepts

- CI: frequent integration, automated builds, automated tests
- CD: Continuous Delivery (manual approval) vs. Continuous Deployment (fully automatic)
- DevOps: culture, practices, tools bridging development and operations
- Pipeline stages: build, unit test, integration test, analysis, deploy, e2e test
- Quality gates: criteria that must pass before proceeding
- Infrastructure as Code (IaC), containerization, monitoring
