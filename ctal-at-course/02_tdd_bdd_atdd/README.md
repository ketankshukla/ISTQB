# 🚀 Chapter 2 — TDD, BDD, and ATDD

## 🎓 Exam Weighting

- **Estimated weight:** ~25% (~10 questions out of 40)
- **K-levels:** K2 (understand), K3 (apply), K4 (analyze)
- **Focus:** TDD cycle, BDD Given/When/Then, ATDD workshops, differences and relationships between the three

---

## 🎯 Learning Objectives

By the end of this chapter, you should be able to:

1. Explain the TDD cycle (Red-Green-Refactor)
2. Describe BDD and write scenarios in Given/When/Then format
3. Explain ATDD and the collaborative workshop approach
4. Compare TDD, BDD, and ATDD and know when to use each
5. Apply TDD/BDD/ATDD to given scenarios

---

## ⭐ Key Concepts

- TDD: Red-Green-Refactor, unit-level, developer-focused
- BDD: Given/When/Then, behavior-focused, business-readable
- ATDD: Collaborative acceptance criteria definition, whole-team
- Relationship: ATDD → BDD → TDD (outer to inner)
- Test pyramid: unit, integration, e2e
