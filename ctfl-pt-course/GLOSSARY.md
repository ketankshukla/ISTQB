# 📖 CTFL-PT Glossary

## ⭐ A

- **Acceptance criteria** — Conditions a system must satisfy to be accepted
- **Anomaly** — Any condition that deviates from expectation

## ⭐ B

- **Baseline** — Reference measurement used for comparison
- **Bottleneck** — Resource that limits overall system performance
- **Breakpoint** — Point at which performance degrades sharply

## ⭐ C

- **Capacity test** — Determines maximum load the system can handle
- **Concurrency** — Number of simultaneous users or processes
- **CPU utilization** — Percentage of CPU capacity in use

## ⭐ E

- **Endurance test** — Long-duration test to detect memory leaks and stability issues
- **Error rate** — Percentage of failed requests

## ⭐ H

- **Hardware utilization** — Measure of CPU, memory, disk, and network usage

## ⭐ L

- **Latency** — Delay between request and first response
- **Load test** — Determines behavior under expected load

## ⭐ P

- **Performance test** — Testing to determine responsiveness, throughput, reliability, and/or scalability
- **Performance indicator** — Metric used to assess system performance
- **Performance profile** — Collection of measurements characterizing performance

## ⭐ R

- **Response time** — Time from request submission to response receipt
- **Resource utilization** — Percentage of system resources consumed

## ⭐ S

- **Scalability** — Ability to handle increased load by adding resources
- **Scalability test** — Determines if system scales with added resources
- **Spike test** — Sudden large increase in load to test reaction
- **Stress test** — Tests beyond normal capacity to find breaking points
- **System under test (SUT)** — The system being tested

## ⭐ T

- **Think time** — Simulated delay between user actions
- **Throughput** — Number of transactions processed per unit of time

## ⭐ V

- **Volume test** — Tests with large amounts of data

## ⭐ W

- **Workload model** — Representation of user activity patterns and intensity
