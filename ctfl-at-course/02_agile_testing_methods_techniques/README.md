## Learning Objectives

By the end of this chapter, you should be able to:

1. Describe how test levels (unit, integration, system, acceptance) are applied in agile
2. Describe how test types (functional, non-functional, regression, confirmation) are applied in agile
3. Explain ATDD and its benefits
4. Explain BDD and its relationship to ATDD and TDD
5. Describe Specification by Example and its workflow
6. Describe exploratory testing in agile, including test charters and SBTM
7. Describe collaboration practices: Three Amigos, pairing, reviewing

---

## Key Concepts to Master

- Test automation pyramid: unit at base, integration in middle, UI at top
- ATDD workflow: Discuss → Distill → Develop → Deliver → Demo
- BDD syntax: Given-When-Then, scenario outlines, backgrounds
- Specification by Example: Using concrete examples instead of abstract statements
- Exploratory testing: charters, time-boxing, session-based management
- Three Amigos: Business + Developer + Tester collaboration

---

## Exam Tips

- This is the highest-weighted chapter. Master ATDD/BDD and exploratory testing.
- Expect scenario-based questions about Given-When-Then scenarios and ATDD workflows.
- Know the test automation pyramid proportions and rationale.
- Distinguish ATDD (acceptance-level, collaborative) from TDD (unit-level, developer-focused).
- Know when to use exploratory testing vs. scripted testing in agile.
