# 🎨 Chapter 3 — Test Automation Architecture and Design

## 🎓 Exam Weighting

- **Estimated weight:** ~20% (~8 questions out of 40)
- **K-levels:** K2, K3
- **Focus:** TAA, framework types, design patterns, tool selection, environment design

---

## 🎯 Learning Objectives

By the end of this chapter, you should be able to:

1. Design a test automation architecture (TAA)
2. Select appropriate framework types
3. Apply design patterns (POM, factory, singleton)
4. Design test data management
5. Plan environment and configuration
6. Select and integrate tools

---

## ⭐ Key Concepts

- TAA (Test Automation Architecture)
- Framework types (linear, modular, data-driven, keyword-driven, hybrid)
- Design patterns (POM, factory, singleton, facade)
- Test data strategies
- Environment configuration
- Tool selection criteria
