# CT-TAS Exam Strategy

## Exam Format

- 40 questions in 60 minutes
- Closed book
- Single best answer (some "choose N")
- 65% pass mark (26/40 correct)

## Time Budget

- 60 minutes / 40 questions = 1.5 minutes per question
- Flag uncertain questions, return at the end
- Reserve 5 minutes for review

## Chapter Weighting

## K-Level Patterns

- **K1 (Remember):** Automation levels, tool types, risk categories
- **K2 (Understand):** ROI calculation, architecture trade-offs, deployment models
- **K3 (Apply):** Design automation strategy for a scenario, calculate ROI, select tools

## Common Distractor Patterns

- **"Test automation" vs "automated testing"** — Strategy vs execution
- **"ROI positive" vs "worth doing"** — ROI may be positive but not the best use of resources
- **"Buy" vs "Build"** — Commercial tools vs custom frameworks

## Pre-Exam Checklist

- [ ] All mock exams completed with 80%+
- [ ] Glossary terms memorized
- [ ] Exam traps reviewed
- [ ] Case studies completed

## Exam Day Advice

- Read each question fully before answering
- Watch for absolutes ("always," "never") — they are usually wrong
- For "choose N" questions, verify you selected exactly N
