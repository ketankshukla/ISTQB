# 🪤 Chapter 6 — Organizational and Human Factors: Exam Traps

## 🪤 Trap 1: Thinking Automation Eliminates Testers

**Trap:** Automation changes tester roles but doesn't eliminate them. Testers become automation engineers, quality coaches, or exploratory testers.

**Example question:**
> Management delays automation because "testers will lose their jobs." The correct response is:
>
> A. They are right — automation replaces testers
> B. Automation evolves roles; testers shift to design, analysis, and complex testing while automation handles repetitive execution
> C. Only hire developers for automation
> D. Only keep manual testing

**Correct answer:** B — Automation augments, not replaces, skilled testers.

---

## 🪤 Trap 2: Hiring Coders Without Testing Knowledge

**Trap:** Pure developers may write technically elegant tests that miss testing principles (coverage, edge cases, risk).

**Example question:**
> A team hires software developers to write automation without testing training. The risk is:
>
> A. No risk — developers write better code
> B. Tests may be technically sound but miss testing principles like coverage, equivalence partitioning, and risk focus
> C. Only slower execution
> D. Only more documentation

**Correct answer:** B — Automation requires both coding AND testing skills.

---

## 🪤 Trap 3: Implementing Automation Without Change Management

**Trap:** Technical solutions fail when people resist. Change management is essential for adoption.

**Example question:**
> A team builds excellent automation but developers refuse to run it in CI. The missing factor is:
>
> A. Better tools
> B. Change management — understanding developer concerns, demonstrating value, and integrating into their workflow
> C. Only more tests
> D. Only faster hardware

**Correct answer:** B — Change management addresses human and process barriers.

---

## 🪤 Trap 4: Reporting Only Technical Metrics to Executives

**Trap:** Executives care about business value (time saved, defects prevented), not technical details (lines of code, test counts).

**Example question:**
> An executive presentation focuses on "500 tests written" and "Selenium Grid configuration." It should instead focus on:
>
> A. Technical architecture
> B. Business outcomes: time saved, defects prevented, faster time-to-market
> C. Only tool costs
> D. Only code coverage percentages

**Correct answer:** B — Executive communication = business value, not technical details.

---

## 🪤 Trap 5: Ignoring Team Satisfaction

**Trap:** Unhappy teams abandon automation. Measure and address team satisfaction.

**Example question:**
> Automation metrics show 90% pass rate and fast execution, but testers complain about maintenance burden. The team should:
>
> A. Ignore complaints — metrics are good
> B. Investigate satisfaction issues; high maintenance may indicate architecture problems
> C. Only hire more testers
> D. Only buy better tools

**Correct answer:** B — Satisfaction metrics reveal sustainability issues that pass rates hide.
