# ⚙️ Chapter 5 — Test Automation Maintenance and Evolution

## 🎓 Exam Weighting

- **Estimated weight:** ~15% (~6 questions out of 40)
- **K-levels:** K2, K3
- **Focus:** Maintenance types, refactoring, version control, evolution, retirement

---

## 🎯 Learning Objectives

By the end of this chapter, you should be able to:

1. Identify types of automation maintenance
2. Refactor automation code effectively
3. Manage test data maintenance
4. Plan for automation evolution
5. Decide when to retire automation assets
6. Manage technical debt in automation

## ⭐ Key Concepts

- Corrective, adaptive, perfective maintenance
- Refactoring patterns
- Version control for tests
- Test data maintenance
- Automation evolution
- Retirement criteria
