# Chapter 1 — Introduction to Test Automation Strategy: Exam Traps

## Trap 1: Confusing Strategy with Execution

**Trap:** Strategy = high-level plan. Execution = writing and running tests. The exam tests strategy thinking.

**Example question:**
> A team writes 500 UI test scripts but has no plan for maintenance, tool selection, or ROI measurement. They have:
>
> A. A complete automation strategy
> B. Automated testing without a strategy; likely unsustainable
> C. The best possible automation approach
> D. Sufficient management support

**Correct answer:** B — Execution without strategy leads to abandonment.

---

## Trap 2: Inverting the Test Pyramid

**Trap:** The pyramid has many unit tests, fewer integration, fewest UI tests. An inverted pyramid (ice cream cone) is an anti-pattern.

**Example question:**
> A team has 50 UI tests, 20 integration tests, and 10 unit tests. This is:
>
> A. A well-balanced test pyramid
> B. An inverted pyramid (anti-pattern) — slow, flaky, expensive to maintain
> C. The ideal ratio for all projects
> D. Only wrong for small projects

**Correct answer:** B — Too many UI tests = inverted pyramid = anti-pattern.

---

## Trap 3: Automating Everything

**Trap:** Not all tests should be automated. Exploratory, one-time, and human-judgment tests remain manual.

**Example question:**
> A manager insists that 100% of all testing must be automated. This is:
>
> A. Achievable and recommended
> B. Incorrect — exploratory testing, usability evaluation, and one-off tests should remain manual
> C. Only wrong for agile projects
> D. Only wrong for regulated industries

**Correct answer:** B — Some testing activities cannot or should not be automated.

---

## Trap 4: Ignoring the Business Case

**Trap:** Automation needs measurable business justification. Without it, projects lose funding.

**Example question:**
> A team starts automation because "everyone is doing it" but cannot show cost savings or quality improvement. The risk is:
>
> A. No risk — automation is always good
> B. Loss of management support and project cancellation when budgets tighten
> C. Only slower test execution
> D. Only more documentation

**Correct answer:** B — Business case with measurable ROI is essential for sustained funding.

---

## Trap 5: Equating Tool Selection with Strategy

**Trap:** Selecting a tool is one part of strategy. Strategy also covers scope, architecture, skills, integration, and metrics.

**Example question:**
> A team selects Selenium as their tool and considers their automation strategy complete. The issue is:
>
> A. Selenium is the wrong tool
> B. Tool selection is only one part; strategy also needs scope, architecture, maintenance plan, and metrics
> C. Only that they need training
> D. Only that they need more licenses

**Correct answer:** B — Strategy = much more than tool selection.
