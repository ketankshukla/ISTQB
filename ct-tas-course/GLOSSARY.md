# 📖 CT-TAS Glossary

## ⭐ A

- **Automated Testing** — The execution of tests using automation tools
- **Automation Framework** — Reusable set of libraries, utilities, and practices for test automation

## ⭐ B

- **Build Verification Test (BVT)** — Automated test suite run on every build to verify basic functionality
- **Business Case** — Document justifying investment in automation with expected costs and benefits

## ⭐ C

- **CI/CD** — Continuous Integration / Continuous Delivery
- **Component Test** — Automated test of a single component in isolation

## ⭐ D

- **Data-Driven Testing** — Executing the same test logic with multiple data sets
- **Deployment Pipeline** — Automated process for building, testing, and deploying software

## ⭐ E

- **End-to-End Test** — Test covering a complete user workflow through the entire system

## ⭐ F

- **False Positive** — Test failure that is not due to a real defect
- **Framework** — Structured automation foundation providing common services

## ⭐ G

- **GUI Testing** — Testing through the graphical user interface
- **Keyword-Driven Testing** — Test design using action keywords in a table format

## ⭐ H

- **Hybrid Framework** — Combines multiple framework approaches (data-driven, keyword, modular)

## ⭐ I

- **Integration Test** — Testing interactions between components or systems

## ⭐ K

- **Keyword** — Action word representing a test operation (e.g., "ClickButton", "EnterText")

## ⭐ L

- **Linear Scripting** — Recording and replaying user actions without abstraction
- **Logical Component** — Reusable test module representing a business function

## ⭐ M

- **Maintainability** — Ease of updating tests when the application changes
- **Model-Based Testing** — Deriving tests from a model of the system behavior

## ⭐ O

- **Object Repository** — Centralized storage of UI element locators

## ⭐ P

- **Page Object Model (POM)** — Design pattern separating page UI from test logic
- **Pilot Project** — Small-scale automation project to validate approach before full adoption

## ⭐ R

- **Record and Playback** — Capturing manual test steps for automated replay
- **Regression Test** — Test verifying that existing functionality still works
- **ROI** — Return on Investment; measure of automation benefit vs cost
- **Reusability** — Ability to use test components across multiple tests

## ⭐ S

- **Scalability** — Ability to expand automation as the application grows
- **Service Virtualization** — Simulating dependent systems for testing
- **Smoke Test** — Quick test verifying that basic functions work
- **Stability** — Consistency of test execution and results

## ⭐ T

- **TAA (Test Automation Architecture)** — High-level design of the automation solution
- **TAF (Test Automation Framework)** — Implementation of the architecture
- **Test Pyramid** — Concept of having more unit tests than integration or UI tests
- **Tooling Strategy** — Selection and integration of automation tools

## ⭐ V

- **Verification and Validation** — Checking that the system meets requirements (verification) and user needs (validation)
