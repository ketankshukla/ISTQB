# Chapter 4 — Dynamic Analysis

## Exam Weighting

- **Estimated weight:** ~10% (~4-5 questions out of 45)
- **K-levels:** K2 (understand), K3 (apply)
- **Focus:** Memory profiling, performance profiling, resource monitoring, runtime behavior analysis

---

## Learning Objectives

By the end of this chapter, you should be able to:

1. Explain the purpose of dynamic analysis
2. Describe memory profiling techniques and detect memory leaks
3. Describe performance profiling and identify bottlenecks
4. Explain code coverage measurement during execution
5. Describe fault injection and runtime monitoring

---

## Key Concepts

- Dynamic analysis: examining code behavior during execution
- Memory profiling: heap usage, memory leaks, allocation patterns
- Performance profiling: CPU usage, method timing, hotspots
- Code coverage: statement, branch, MC/DC measured during execution
- Fault injection: introducing errors to test resilience
- Runtime monitoring: APM, logging, alerting
