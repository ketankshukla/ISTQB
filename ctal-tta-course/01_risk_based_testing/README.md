# Chapter 1 — Risk-Based Testing (Technical Perspective)

## Exam Weighting

- **Estimated weight:** ~10% (~4-5 questions out of 45)
- **K-levels:** K2 (understand), K3 (apply)
- **Focus:** Technical risks, selecting white-box techniques based on risk, risk assessment from a technical tester perspective

---

## Learning Objectives

By the end of this chapter, you should be able to:

1. Explain how risk-based testing applies from a technical test analyst perspective
2. Identify technical risks in software systems
3. Select appropriate white-box and static analysis techniques based on identified risks
4. Explain how technical test analysts contribute to risk identification and mitigation

---

## Key Concepts

- Technical risks: code complexity, technology, security, performance
- Risk identification techniques: FMEA, fault tree analysis, checklists
- Risk-based technique selection: matching techniques to risk types
- Technical test analyst role in risk-based testing
