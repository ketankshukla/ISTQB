# 🎯 Chapter 2 — White-Box Test Techniques

## 🎓 Exam Weighting

- **Estimated weight:** ~25% (~11-12 questions out of 45)
- **K-levels:** K2 (understand), K3 (apply), K4 (analyze)
- **Focus:** Control flow testing, data flow testing, code coverage, cyclomatic complexity, MC/DC

---

## 🎯 Learning Objectives

By the end of this chapter, you should be able to:

1. Explain control flow testing and draw control flow graphs
2. Calculate statement, branch, and path coverage
3. Explain data flow testing and identify definition-use pairs
4. Calculate cyclomatic complexity and interpret its meaning
5. Explain MC/DC (Modified Condition/Decision Coverage) and when it is required
6. Apply white-box techniques to evaluate test coverage

---

## ⭐ Key Concepts

- Control flow graphs (CFG): nodes, edges, decision nodes, junction nodes
- Statement coverage: every statement executed at least once
- Branch/decision coverage: every decision outcome (true/false) executed at least once
- Path coverage: every path through the code executed
- Cyclomatic complexity: E - N + 2P
- Data flow testing: definitions, uses, definition-use (DU) pairs
- MC/DC: each condition independently affects the decision outcome
