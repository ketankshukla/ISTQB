# 🛠️ Chapter 5 — Test Tools and Automation

## 🎓 Exam Weighting

- **Estimated weight:** ~15% (~7 questions out of 45)
- **K-levels:** K2 (understand), K3 (apply)
- **Focus:** Tool selection, test automation frameworks, CI/CD integration, tool metrics, specific tools for technical testing

---

## 🎯 Learning Objectives

By the end of this chapter, you should be able to:

1. Describe tool categories for technical testing
2. Explain criteria for selecting technical test tools
3. Describe test automation frameworks and their components
4. Explain CI/CD integration for automated testing
5. Describe tool metrics and success criteria

---

## ⭐ Key Concepts

- Tool categories: unit test frameworks, coverage tools, static analysis, dynamic analysis, performance, security
- Test automation frameworks: keyword-driven, data-driven, hybrid, BDD
- CI/CD integration: automated build, test, report
- Tool selection criteria: language support, integration, cost, learning curve
- Metrics: coverage trends, execution time, defect detection rate
