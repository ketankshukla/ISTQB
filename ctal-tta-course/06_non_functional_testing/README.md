# Chapter 6 — Non-Functional Testing

## Exam Weighting

- **Estimated weight:** ~25% (~11-12 questions out of 45)
- **K-levels:** K2 (understand), K3 (apply), K4 (analyze)
- **Focus:** Performance, security, reliability, compatibility, usability testing

---

## Learning Objectives

By the end of this chapter, you should be able to:

1. Explain performance testing types (load, stress, scalability, soak)
2. Describe security testing techniques (SAST, DAST, penetration testing)
3. Explain reliability testing (availability, recovery, failover)
4. Describe compatibility testing approaches
5. Explain usability testing considerations for technical testers

---

## Key Concepts

- Performance: load, stress, soak, spike, scalability testing
- Security: SAST, DAST, penetration testing, vulnerability scanning
- Reliability: MTBF, MTTR, availability, failover testing
- Compatibility: browser, OS, device, backward compatibility
- Usability: accessibility, user experience, error handling
