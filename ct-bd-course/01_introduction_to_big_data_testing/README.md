# Chapter 1 — Introduction to Big Data Testing

## Exam Weighting

- **Estimated weight:** ~15% (~6 questions out of 40)
- **K-levels:** K1 (remember), K2 (understand)
- **Focus:** Big Data characteristics, differences from traditional testing, challenges

---

## Learning Objectives

By the end of this chapter, you should be able to:

1. Explain the 5 Vs of Big Data
2. Describe differences between Big Data testing and traditional testing
3. Identify challenges specific to Big Data testing
4. Recognize structured, semi-structured, and unstructured data
5. Describe the role of testing in the Big Data lifecycle

---

## Key Concepts

- Volume, Velocity, Variety, Veracity, Value
- Structured vs. semi-structured vs. unstructured data
- Data lakes vs. data warehouses
- Testing challenges: scale, schema flexibility, distributed processing
- Shift-left testing for Big Data
