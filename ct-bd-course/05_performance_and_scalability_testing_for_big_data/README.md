# ⚡ Chapter 5 — Performance and Scalability Testing for Big Data

## 🎓 Exam Weighting

- **Estimated weight:** ~15% (~6 questions out of 40)
- **K-levels:** K2 (understand), K3 (apply)
- **Focus:** Benchmarks, throughput, latency, cluster scaling, data skew

---

## 🎯 Learning Objectives

By the end of this chapter, you should be able to:

1. Describe performance testing objectives for Big Data systems
2. Define throughput and latency metrics
3. Identify bottlenecks in distributed processing
4. Test horizontal and vertical scaling
5. Detect and mitigate data skew

---

## ⭐ Key Concepts

- Throughput (records/second, GB/hour)
- Latency (ingestion latency, query latency)
- Benchmarks (TPC-DS, HiBench)
- Bottlenecks (CPU, I/O, network, shuffle)
- Data skew and hot spots
- Horizontal vs. vertical scaling
