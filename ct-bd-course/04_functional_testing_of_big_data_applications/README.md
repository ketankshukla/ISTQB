# Chapter 4 — Functional Testing of Big Data Applications

## Exam Weighting

- **Estimated weight:** ~20% (~8 questions out of 40)
- **K-levels:** K2 (understand), K3 (apply)
- **Focus:** ETL/ELT testing, transformation logic, report validation, data lineage

---

## Learning Objectives

By the end of this chapter, you should be able to:

1. Test ETL and ELT pipelines for correctness
2. Validate transformation rules and business logic
3. Test MapReduce and Spark job outputs
4. Validate BI reports and dashboards
5. Trace data lineage end-to-end

---

## Key Concepts

- ETL vs. ELT testing
- Source-to-target mapping validation
- Transformation rule testing
- Incremental vs. full load testing
- Report reconciliation
- Data lineage tracking
