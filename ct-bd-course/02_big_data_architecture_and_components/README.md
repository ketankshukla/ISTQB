# 🗃️ Chapter 2 — Big Data Architecture and Components

## 🎓 Exam Weighting

- **Estimated weight:** ~15% (~6 questions out of 40)
- **K-levels:** K2 (understand), K3 (apply)
- **Focus:** Hadoop/Spark components, data flow, architecture layers

---

## 🎯 Learning Objectives

By the end of this chapter, you should be able to:

1. Describe the Hadoop ecosystem components (HDFS, MapReduce, YARN)
2. Explain Apache Spark and its role in Big Data processing
3. Identify the layers of Big Data architecture (ingestion, storage, processing, consumption)
4. Describe NoSQL databases and their use cases
5. Apply testing approaches for each architectural layer

---

## ⭐ Key Concepts

- Hadoop Distributed File System (HDFS)
- MapReduce paradigm
- YARN resource management
- Apache Spark (RDDs, DataFrames, Spark SQL)
- Data ingestion tools (Sqoop, Flume, Kafka)
- NoSQL types (key-value, document, column-family, graph)
- Data flow from source to consumption
