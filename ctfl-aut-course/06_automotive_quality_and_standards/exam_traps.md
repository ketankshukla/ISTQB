# 🪤 Chapter 6 — Automotive Quality and Standards: Exam Traps

## 🪤 Trap 1: ISO 26262 is Only About Software

**Trap:** ISO 26262 covers system, hardware, and software; Part 6 is software-specific.

**Example question:**
> ISO 26262 only applies to software development. This is:
>
> A. Correct — it's a software standard
> B. Incorrect — ISO 26262 covers system-level (Part 4), hardware (Part 5), and software (Part 6) development
> C. Only for ADAS
> D. Only for powertrain

**Correct answer:** B — ISO 26262 = system + hardware + software.

---

## 🪤 Trap 2: ASPICE is a Test Standard

**Trap:** ASPICE is a process assessment model covering all development, not just testing.

**Example question:**
> ASPICE is a testing standard that defines test techniques. This is:
>
> A. Correct — it's about testing
> B. Incorrect — ASPICE is a process assessment model covering all development processes (requirements, design, coding, testing, management), not just testing
> C. Only for German manufacturers
> D. Only for suppliers

**Correct answer:** B — ASPICE = process model, not test standard.

---

## 🪤 Trap 3: AUTOSAR Eliminates Testing

**Trap:** AUTOSAR standardizes architecture but doesn't eliminate the need for testing.

**Example question:**
> AUTOSAR-compliant software doesn't need testing because the architecture is standardized. This is:
>
> A. Correct — standardization ensures quality
> B. Incorrect — AUTOSAR standardizes interfaces and architecture but application logic still requires full testing; standardization may improve testability but doesn't eliminate testing
> C. Only for BSW
> D. Only for RTE

**Correct answer:** B — AUTOSAR ≠ no testing needed.

---

## 📊 Trap 4: High Coverage Guarantees Quality

**Trap:** Coverage measures test thoroughness, not correctness or absence of defects.

**Example question:**
> 100% statement coverage guarantees the software is defect-free. This is:
>
> A. Correct — every statement was executed
> B. Incorrect — coverage measures which code was executed, not whether it behaves correctly; defects in logic, timing, and requirements can still exist
> C. Only for unit tests
> D. Only for system tests

**Correct answer:** B — Coverage ≠ defect-free.

---

## 🪤 Trap 5: Audits Only Check Documents

**Trap:** Audits check process adherence, evidence, and actual practices, not just paperwork.

**Example question:**
> Passing an audit only requires having the correct documents in place. This is:
>
> A. Correct — documentation is what auditors check
> B. Incorrect — audits verify process adherence, evidence of execution, traceability, and actual practices, not just the existence of documents
> C. Only for internal audits
> D. Only for external audits

**Correct answer:** B — Audits = process + evidence + practice.
