# Chapter 4 — Security Testing Process: Exam Traps

## Trap 1: Security Testing Needs No Planning

**Trap:** Security testing requires formal planning with scope, objectives, and constraints.

**Example question:**
> Security testing can be performed ad-hoc without formal planning. This is:
>
> A. Correct — security testing is exploratory
> B. Incorrect — security testing requires planning: scope definition, objective setting, technique selection, and constraint identification
> C. Only for small projects
> D. Only for agile teams

**Correct answer:** B — Planning is essential.

---

## Trap 2: All Findings are Critical

**Trap:** Findings must be risk-assessed; not everything is critical.

**Example question:**
> Every security finding must be fixed immediately. This is:
>
> A. Correct — all security issues are serious
> B. Incorrect — findings should be risk-assessed; low-risk issues can be scheduled for future releases
> C. Only for financial systems
> D. Only for healthcare

**Correct answer:** B — Risk-based prioritization.

---

## Trap 3: Security Reports Only List Vulnerabilities

**Trap:** Reports need evidence, risk ratings, and remediation steps.

**Example question:**
> A security test report only needs to list the vulnerabilities found. This is:
>
> A. Correct — the list is the deliverable
> B. Incorrect — reports need evidence (screenshots, logs), risk ratings, and specific remediation recommendations to be actionable
> C. Only for internal teams
> D. Only for external auditors

**Correct answer:** B — Reports need context and action.

---

## Trap 4: Security Testing is Separate from Main Test Process

**Trap:** Security testing should be integrated throughout the test lifecycle.

**Example question:**
> Security testing is performed by a separate team after all other testing is complete. This is:
>
> A. Correct — security is a specialist activity
> B. Incorrect — security should be integrated throughout: requirements, design, development, and testing phases
> C. Only for waterfall
> D. Only for regulated industries

**Correct answer:** B — Integrate throughout lifecycle.

---

## Trap 5: CVSS Score is the Only Risk Measure

**Trap:** CVSS provides a base score; environmental factors also matter.

**Example question:**
> A vulnerability's CVSS base score alone determines its priority for remediation. This is:
>
> A. Correct — CVSS is the industry standard
> B. Incorrect — while CVSS base score is important, environmental factors (exposure, asset value, mitigations) also affect actual risk priority
> C. Only for open source
> D. Only for commercial software

**Correct answer:** B — Environmental factors matter too.
