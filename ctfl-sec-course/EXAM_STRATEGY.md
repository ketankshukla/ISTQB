# ♟️ CTFL-SEC Exam Strategy

## 🎓 Exam Format

- 20 questions in 40 minutes
- Closed book
- Single best answer
- 65% pass mark (13/20 correct)

## ⭐ Time Budget

- 40 minutes / 20 questions = 2 minutes per question
- Flag uncertain questions, return at the end
- Reserve 5 minutes for review

## ⭐ Chapter Weighting

| Chapter | Est. Questions | Priority |
|---------|---------------|----------|
| 2 — Security Testing Principles | ~4 | High |
| 3 — Security Testing Techniques | ~4 | High |
| 1 — Introduction | ~3 | Medium |
| 4 — Security Testing Process | ~3 | Medium |
| 5 — Security Testing Tools | ~3 | Medium |
| 6 — Security Testing in the SDLC | ~3 | Medium |

## 🪜 K-Level Patterns

- **K1 (Remember):** Security terms, OWASP categories, threat types
- **K2 (Understand):** How vulnerabilities arise; when to apply techniques
- **K3 (Apply):** Select appropriate test technique for a vulnerability

## ⭐ Common Distractor Patterns

- **"Security testing is only penetration testing"** — False; includes SAST, DAST, code review, risk analysis
- **"Only security experts can do security testing"** — False; testers can use tools and checklists
- **"Security testing happens at the end"** — False; must be integrated throughout SDLC

## 🎓 Pre-Exam Checklist

- [ ] All mock exams completed with 80%+
- [ ] Glossary terms memorized
- [ ] Exam traps reviewed
- [ ] Case studies completed

## 🎓 Exam Day Advice

- Read each question fully before answering
- Watch for absolutes ("always," "never")
- Remember: security testing is part of all testers' responsibilities
