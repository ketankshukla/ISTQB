# Chapter 2 — Scaling Agile Testing

## Exam Weighting

- **Estimated weight:** ~20% (~8 questions out of 40)
- **K-levels:** K2, K3
- **Focus:** Multi-team testing, integration, environments, coordination, DoD

---

## Learning Objectives

By the end of this chapter, you should be able to:

1. Design testing approaches for multiple agile teams
2. Coordinate integration testing across teams
3. Manage shared test environments
4. Define and maintain Definition of Done at scale
5. Handle dependencies between teams
6. Plan regression testing across teams

---

## Key Concepts

- Multi-team test planning
- Integration and system testing at scale
- Environment management
- Definition of Done
- Dependency management
- Regression strategy at scale
