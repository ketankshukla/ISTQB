# CT-ATLaS Exam Strategy

## Exam Format

- 40 questions in 60 minutes
- Closed book
- Single best answer (some "choose N")
- 65% pass mark (26/40 correct)

## Time Budget

- 60 minutes / 40 questions = 1.5 minutes per question
- Flag uncertain questions, return at the end
- Reserve 5 minutes for review

## K-Level Patterns

- **K1 (Remember):** SAFe, LeSS, Spotify model terms; agile scaling frameworks
- **K2 (Understand):** Why coordination is needed; how metrics drive behavior
- **K3 (Apply):** Design test approach for scaled agile; select metrics for a scenario

## Common Distractor Patterns

- **"Agile means no planning"** — False; scaled agile requires more coordination
- **"Centralized QA" vs "embedded QA"** — Different models with different trade-offs
- **"Velocity = productivity"** — False; velocity is a planning tool, not a quality metric

## Pre-Exam Checklist

- [ ] All mock exams completed with 80%+
- [ ] Glossary terms memorized
- [ ] Exam traps reviewed
- [ ] Case studies completed

## Exam Day Advice

- Read each question fully before answering
- Watch for absolutes ("always," "never") — they are usually wrong
- For "choose N" questions, verify you selected exactly N
