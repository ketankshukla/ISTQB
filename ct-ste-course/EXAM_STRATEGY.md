# CT-STE Exam Strategy

## Exam Format

- 40 questions in 60 minutes
- Closed book
- Single best answer (some "choose N")
- 65% pass mark (26/40 correct)

## Time Budget

- 60 minutes / 40 questions = 1.5 minutes per question
- Flag uncertain questions, return at the end
- Reserve 5 minutes for review

## Chapter Weighting

| Chapter | Est. Questions | Priority |
|---------|---------------|----------|
| 2 — Security Testing Planning and Strategy | ~8 | High |
| 3 — Security Testing Techniques | ~8 | High |
| 5 — Security Test Execution and Analysis | ~8 | High |
| 4 — Security Test Automation | ~6 | Medium |
| 6 — Security Testing in the SDLC | ~6 | Medium |
| 1 — Introduction | ~4 | Low |

## K-Level Patterns

- **K1 (Remember):** OWASP Top 10, CIA triad, security testing terms
- **K2 (Understand):** When to use SAST vs DAST, risk assessment
- **K3 (Apply):** Design security tests for a scenario, select tools

## Common Distractor Patterns

- **"Security testing is only for specialists"** — False; security is everyone's responsibility
- **"Penetration testing = all security testing"** — False; security testing is broader
- **"Security testing happens at the end"** — False; shift-left security testing

## Pre-Exam Checklist

- [ ] All mock exams completed with 80%+
- [ ] Glossary terms memorized
- [ ] Exam traps reviewed
- [ ] Case studies completed

## Exam Day Advice

- Read each question fully before answering
- Watch for absolutes ("always," "never") — they are usually wrong
- For "choose N" questions, verify you selected exactly N
