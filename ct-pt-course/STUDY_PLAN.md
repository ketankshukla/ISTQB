## Study Techniques That Work

**1. Active Recall**
After reading a section, close the file and explain the concept aloud as if teaching a colleague.

**2. Spaced Repetition**
Re-visit practice questions from previous chapters every 3-4 days.

**3. Compare and Contrast**
Create a comparison table for test types (load vs. stress vs. endurance vs. spike) and KPIs.

**4. Build Scenarios**
For each test type, create a small scenario: "If I need to test X, I should use Y because..."

**5. Work the Numbers**
Practice calculating response times, throughput, and resource utilization from sample data.

## Exam Day Checklist

- [ ] 7+ hours of sleep the night before
- [ ] Review `exam_traps.md` one final time
- [ ] Arrive 15 minutes early (or set up exam space)
- [ ] Read each question twice before selecting an answer
- [ ] Flag uncertain questions and return to them
- [ ] Budget ~1.5 minutes per question (leaves 10 minutes for review)
- [ ] Trust your first instinct unless you find clear evidence to change
