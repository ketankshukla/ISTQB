# Chapter 3 — Performance Test Planning and Design

## Exam Weighting

- **Estimated weight:** ~20% (~8 questions out of 40)
- **K-levels:** K2 (understand), K3 (apply)
- **Focus:** Strategy, workloads, scenarios, KPIs, test data, environment sizing

---

## Learning Objectives

By the end of this chapter, you should be able to:

1. Define a performance test strategy
2. Create workload models and user profiles
3. Define performance scenarios and transaction mixes
4. Identify and define KPIs and acceptance criteria
5. Plan test data and environment requirements

---

## Key Concepts

- Performance test strategy and scope
- Workload modeling and user profiles
- Transaction mix and think time
- KPIs: response time, throughput, utilization, error rate
- SLAs and acceptance criteria
- Test data requirements
- Environment sizing and representativeness
