# Chapter 4 — Performance Test Execution and Monitoring

## Exam Weighting

- **Estimated weight:** ~15% (~6 questions out of 40)
- **K-levels:** K2 (understand), K3 (apply)
- **Focus:** Test phases, ramp-up, monitoring, bottleneck detection

---

## Learning Objectives

By the end of this chapter, you should be able to:

1. Describe the phases of performance test execution
2. Explain ramp-up and ramp-down strategies
3. Identify what to monitor during test execution
4. Describe common bottlenecks and their indicators
5. Understand test execution constraints and prerequisites

---

## Key Concepts

- Test phases: initialization, ramp-up, steady state, ramp-down, teardown
- Resource monitoring: CPU, memory, disk, network
- Application monitoring: database, thread pools, connection pools
- Bottleneck indicators
- Test execution constraints
