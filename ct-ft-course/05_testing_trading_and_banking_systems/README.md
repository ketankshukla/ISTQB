# Chapter 5 — Testing Trading and Banking Systems

## Exam Weighting

- **Estimated weight:** ~20% (~8 questions out of 40)
- **K-levels:** K2, K3
- **Focus:** OMS/EMS, algorithmic trading, payment testing, reconciliation, settlement

---

## Learning Objectives

By the end of this chapter, you should be able to:

1. Test order management and execution systems
2. Validate algorithmic trading strategies
3. Test payment processing workflows
4. Verify reconciliation and settlement processes
5. Test corporate actions processing
6. Design end-to-end tests for financial workflows

---

## Key Concepts

- Order lifecycle (new, modify, cancel, fill, partial fill)
- Algorithmic trading strategies (VWAP, TWAP, iceberg)
- Payment processing (authorization, clearing, settlement)
- Reconciliation types (internal, external, nostro/vostro)
- Settlement cycles (T+1, T+2)
- Corporate actions (dividends, splits, mergers)
- End-to-end testing approaches
