# Chapter 1 — Introduction to Finance Testing

## Exam Weighting

- **Estimated weight:** ~10% (~4 questions out of 40)
- **K-levels:** K1, K2
- **Focus:** Finance testing objectives, scope, terminology, types of financial systems

---

## Learning Objectives

By the end of this chapter, you should be able to:

1. Define finance testing and its objectives
2. Identify types of financial systems (banking, trading, insurance, payments)
3. Describe the unique characteristics of financial software
4. Explain the role of testing in financial services
5. Identify stakeholders in finance testing

---

## Key Concepts

- Finance testing definition and objectives
- Types of financial systems
- Unique characteristics of financial software (precision, auditability, compliance)
- Finance testing vs general software testing
- Stakeholders (regulators, auditors, traders, risk managers)
