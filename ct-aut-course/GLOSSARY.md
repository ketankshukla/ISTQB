# 📖 CT-AuT Glossary

## ⭐ A

- **ASIL (Automotive Safety Integrity Level)** — Risk classification from A (lowest) to D (highest) per ISO 26262
- **ASPICE (Automotive SPICE)** — Process assessment model for automotive software development
- **AUTOSAR** — Standardized automotive software architecture

## ⭐ C

- **CAN (Controller Area Network)** — Vehicle bus standard for communication between ECUs
- **CAPL** — CANoe programming language for test automation
- **C-TP (Confirmation Test Procedure)** — Test to confirm a fix works

## ⭐ D

- **DTC (Diagnostic Trouble Code)** — Code indicating a vehicle fault
- **Diagnostic** — Process of identifying vehicle system faults

## ⭐ E

- **ECU (Electronic Control Unit)** — Embedded computer controlling vehicle functions
- **E2E (End-to-End)** — Communication protection ensuring data integrity

## ⭐ F

- **FMEA (Failure Mode and Effects Analysis)** — Systematic method for identifying potential failures
- **FTTI (Fault Tolerant Time Interval)** — Time between fault occurrence and hazardous event

## ⭐ H

- **HARA (Hazard Analysis and Risk Assessment)** — Method to identify hazards and assess risk per ISO 26262
- **HW/SW Interface** — Specification defining how software interacts with hardware

## ⭐ I

- **ISO 26262** — Functional safety standard for road vehicles
- **Integration Testing** — Testing combined software and hardware components

## ⭐ L

- **LIN (Local Interconnect Network)** — Low-cost vehicle bus for door locks, seats, etc.

## ⭐ M

- **MC/DC (Modified Condition/Decision Coverage)** — Coverage criterion required for high ASIL levels
- **MIL (Model in the Loop)** — Testing using software models

## ⭐ N

- **Network Management** — Protocol for coordinated ECU sleep/wake behavior

## ⭐ O

- **OTA (Over-the-Air)** — Wireless software updates to vehicles
- **OSEK/VDX** — Operating system standard for automotive ECUs

## ⭐ P

- **PIL (Processor in the Loop)** — Testing on target processor with simulated environment
- **Process Assessment** — Evaluating development process maturity (ASPICE)

## ⭐ R

- **Regression Testing** — Re-testing after changes to ensure no new defects
- **Requirements Traceability** — Linking requirements to tests and code

## ⭐ S

- **Safety Goal** — Top-level safety requirement from ISO 26262
- **SIL (Software in the Loop)** — Testing using software simulation
- **SWC (Software Component)** — AUTOSAR reusable software unit

## ⭐ T

- **Traceability** — Ability to follow requirements through design, code, and test

## ⭐ U

- **UDS (Unified Diagnostic Services)** — Protocol for vehicle diagnostics

## ⭐ V

- **V-Model** — Development lifecycle with corresponding test phases
- **Vehicle Network** — Communication system connecting ECUs (CAN, LIN, FlexRay, Ethernet)

## ⭐ X

- **XIL (Everything in the Loop)** — Umbrella term for MIL, SIL, PIL, HIL
