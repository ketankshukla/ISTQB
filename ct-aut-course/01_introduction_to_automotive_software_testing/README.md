# Chapter 1 — Introduction to Automotive Software Testing

## Exam Weighting

- **Estimated weight:** ~10% (~4 questions out of 40)
- **K-levels:** K1, K2
- **Focus:** Automotive domain, ECUs, vehicle networks, safety-critical testing

---

## Learning Objectives

By the end of this chapter, you should be able to:

1. Describe the automotive software landscape
2. Explain the role of ECUs and vehicle networks
3. Identify characteristics of safety-critical automotive software
4. Describe the V-model in automotive
5. Recognize the importance of traceability

---

## Key Concepts

- ECUs and vehicle architecture
- CAN, LIN, FlexRay, Ethernet
- Safety-critical software characteristics
- Automotive V-model
- Traceability requirements
