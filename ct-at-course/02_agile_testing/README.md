# 🏃 Chapter 2 — Agile Testing

## 🎓 Exam Weighting

- **Estimated weight:** ~25% (~10 questions out of 40)
- **K-levels:** K2 (understand), K3 (apply)
- **Focus:** Whole-team approach, tester role, acceptance criteria, Definition of Done, test planning

---

## 🎯 Learning Objectives

By the end of this chapter, you should be able to:

1. Explain the whole-team approach to quality
2. Describe the role of the tester in agile
3. Explain acceptance criteria and how they drive testing
4. Describe the Definition of Done and its testing implications
5. Explain test planning in agile

---

## ⭐ Key Concepts

- Whole-team approach
- Tester's role: collaborator, coach, automator, explorer
- Acceptance criteria and testable stories
- Definition of Done including testing
- Agile test planning
