### Q20 (AT-3.2, K2)

---

## Answers and Rationales

### Q1
**Answer: B**

**Rationale:** Exploratory testing = simultaneous learning, test design, and execution.

### Q2
**Answer: B**

**Rationale:** SBTM uses charters (mission), time-boxes (duration), and debriefs (review).

### Q3
**Answer: B**

**Rationale:** BDD's primary focus is collaboration and shared understanding through scenarios.

### Q4
**Answer: B**

**Rationale:** Given/When/Then is the standard BDD scenario format.

### Q5
**Answer: B**

**Rationale:** ATDD = acceptance tests written before implementation to drive development.

### Q6
**Answer: B**

**Rationale:** TDD cycle = Red (failing test), Green (passing code), Refactor (improve design).

### Q7
**Answer: B**

**Rationale:** Pyramid = many unit (base), some integration, few UI (top).

### Q8
**Answer: C**

**Rationale:** Exploratory testing = Q3 (business-facing, critique the product).

### Q9
**Answer: B**

**Rationale:** Unexpected behavior leading to investigation = learning and adaptation in exploratory testing.

### Q10
**Answer: B**

**Rationale:** Three Amigos = business, developer, and tester collaborating on scenarios.

### Q11
**Answer: B**

**Rationale:** Red step = write a failing test before any implementation exists.

### Q12
**Answer: B**

**Rationale:** Tests before coding that guide implementation = ATDD.

### Q13
**Answer: B**

**Rationale:** Unit tests are primarily written by developers, often using TDD.

### Q14
**Answer: D**

**Rationale:** Performance testing = Q4 (technology-facing, critique the product).

### Q15
**Answer: B**

**Rationale:** A charter includes mission, approach, and duration.

### Q16
**Answer: B**

**Rationale:** BDD scenarios serve as living documentation describing expected behavior.

### Q17
**Answer: B**

**Rationale:** In TDD, the test fails first because the code does not yet exist.

### Q18
**Answer: B**

**Rationale:** Brian Marick created the Agile Testing Quadrants.

### Q19
**Answer: B**

**Rationale:** 300 UI tests with 30 unit tests = inverted pyramid (ice cream cone).

### Q20
**Answer: B**

**Rationale:** Given/When/Then is readable by technical and non-technical stakeholders.
