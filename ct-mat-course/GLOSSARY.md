# 📖 CT-MAT Glossary

## ⭐ A

- **APK (Android Application Package)** — File format used to distribute and install Android apps
- **App Store** — Apple's platform for distributing iOS apps (also used generically for app marketplaces)
- **APK Analyzer** — Tool to inspect APK contents and identify optimization opportunities

## ⭐ B

- **Beta Testing** — Distributing pre-release versions to external users for real-world feedback
- **Build Flavor** — Variant of an app (e.g., free vs paid, staging vs production)

## ⭐ C

- **Certificate Pinning** — Hard-coding server certificate to prevent MITM attacks
- **Content Provider** — Android component for sharing data between apps
- **CRUD** — Create, Read, Update, Delete — basic data operations

## ⭐ D

- **Dalvik/ART** — Android runtime environments (ART replaced Dalvik in Android 5.0)
- **Deep Link** — URL that opens specific content within an app
- **Device Fragmentation** — Wide variety of device types, screen sizes, OS versions

## ⭐ E

- **Emulator** — Software mimicking device hardware (Android Emulator)
- **Entitlements** — Permissions or features unlocked for a user (common in in-app purchases)

## ⭐ F

- **Fragment** — Android UI component representing a portion of activity behavior
- **Flutter** — Google's cross-platform UI framework (Dart)

## ⭐ G

- **Gesture** — Touch interaction (swipe, pinch, long-press, double-tap)
- **GPS Mocking** — Simulating location data for testing location-based features

## ⭐ I

- **IPA (iOS App Store Package)** — File format for iOS app distribution
- **Intent** — Android messaging object for requesting action from another component

## ⭐ J

- **Jailbreak** — Removing iOS software restrictions to gain root access
- **Jetpack Compose** — Modern Android UI toolkit

## ⭐ L

- **Lifecycle** — App states: foreground, background, suspended, terminated
- **Location Services** — GPS, Wi-Fi, cellular triangulation for positioning

## ⭐ M

- **MDM (Mobile Device Management)** — Enterprise software to manage mobile devices
- **Mobile Backend as a Service (MBaaS)** — Cloud services for mobile apps (push, auth, storage)

## ⭐ N

- **Native App** — Built specifically for one platform using platform language (Swift, Kotlin)
- **Network Throttling** — Simulating slow or intermittent network conditions

## ⭐ O

- **OTA (Over-The-Air)** — Wireless delivery of updates

## ⭐ P

- **Permissions** — User-granted access to device features (camera, location, contacts)
- **ProGuard/R8** — Android code shrinking and obfuscation tools
- **Push Notification** — Message sent from server to device

## ⭐ R

- **React Native** — Cross-platform framework using JavaScript/React
- **Rooting** — Gaining root access on Android (similar to jailbreak)
- **Rotation** — Switching between portrait and landscape orientations

## ⭐ S

- **Simulator** — Software mimicking device software behavior (iOS Simulator)
- **Splash Screen** — Initial screen shown while app loads
- **Store Compliance** — Meeting Apple/Google guidelines for app store approval

## ⭐ T

- **TestFlight** — Apple's platform for beta testing iOS apps
- **Throttling** — Limiting network speed for testing purposes

## ⭐ U

- **Universal Link / App Link** — Standard web URLs that open in app instead of browser
- **USB Debugging** — Android feature allowing development tools to communicate with device

## ⭐ V

- **View Controller** — iOS component managing a portion of the user interface
- **Virtual Device** — Software-created mobile device for testing

## ⭐ W

- **WebView** — In-app browser component for displaying web content
- **Widget** — Home screen component showing app content without opening the app

## ⭐ X

- **Xcode** — Apple's IDE for iOS/macOS development
- **XCUITest** — Apple's UI testing framework for iOS
