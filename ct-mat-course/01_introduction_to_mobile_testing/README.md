# Chapter 1 — Introduction to Mobile Testing

## Exam Weighting

- **Estimated weight:** ~10% (~4 questions out of 40)
- **K-levels:** K1, K2
- **Focus:** Mobile testing objectives, challenges, types, SDLC integration

---

## Learning Objectives

By the end of this chapter, you should be able to:

1. Define mobile application testing and its objectives
2. Describe unique challenges of mobile testing
3. Identify types of mobile apps (native, web, hybrid)
4. Explain the mobile SDLC
5. Identify stakeholders in mobile testing

---

## Key Concepts

- Mobile testing definition and objectives
- Native, hybrid, and web apps
- Device fragmentation and OS versions
- Mobile SDLC (design, develop, test, deploy, maintain)
- App store submission and approval
- Mobile testing stakeholders
