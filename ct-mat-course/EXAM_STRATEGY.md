# CT-MAT Exam Strategy

## Exam Format

- 40 questions in 60 minutes
- Closed book
- Single best answer (some "choose N")
- 65% pass mark (26/40 correct)

## Time Budget

- 60 minutes / 40 questions = 1.5 minutes per question
- Flag uncertain questions, return at the end
- Reserve 5 minutes for review

## K-Level Patterns

- **K1 (Remember):** Mobile OS versions, device types, app store requirements
- **K2 (Understand):** Why fragmentation matters, when to use emulators vs real devices
- **K3 (Apply):** Select test devices for coverage, design interruption tests, plan automation strategy

## Common Distractor Patterns

- **"Emulator" vs "Simulator" vs "Real device"** — Emulators mimic hardware; simulators mimic behavior; real devices give true user experience
- **"Functional" vs "Non-functional"** — UI testing is functional; battery drain is non-functional
- **"iOS" vs "Android" differences** — Different update models, approval processes, hardware diversity

## Pre-Exam Checklist

- [ ] All mock exams completed with 80%+
- [ ] Glossary terms memorized
- [ ] Exam traps reviewed
- [ ] Case studies completed

## Exam Day Advice

- Read each question fully before answering
- Watch for absolutes ("always," "never") — they are usually wrong
- For "choose N" questions, verify you selected exactly N
