# Chapter 3 — Usability Evaluation Methods: Exam Traps

## Trap 1: Thinking Expert Methods Replace User Testing

**Trap:** Expert methods are faster and cheaper but do not capture real user behavior, emotions, or satisfaction. They complement, not replace, user testing.

**Example question:**
> A team conducts a thorough heuristic evaluation and declares the product ready for release without any user testing. This is:
>
> A. Appropriate — heuristic evaluation is sufficient
> B. Risky — expert methods cannot validate real user behavior, emotions, or satisfaction
> C. Faster than including user testing
> D. Required by ISO standards

**Correct answer:** B — Expert methods find problems but cannot validate actual user experience. User testing is needed.

---

## Trap 2: Confusing Cognitive Walkthrough and Heuristic Evaluation

**Trap:** Cognitive walkthrough focuses on learnability by simulating a user's problem-solving step-by-step. Heuristic evaluation checks the interface against general principles.

**Example question:**
> A usability expert simulates a first-time user's problem-solving process at each step of a task to evaluate learnability. This is:
>
> A. Heuristic evaluation
> B. Cognitive walkthrough
> C. Think-aloud protocol
> D. A/B testing

**Correct answer:** B — Simulating a user's step-by-step problem-solving = cognitive walkthrough. Heuristic evaluation checks against general principles.

---

## Trap 3: Thinking One Method Is Always Best

**Trap:** The best method depends on the situation: budget, timeline, stage of design, and what you need to learn.

**Example question:**
> A team needs to measure task completion time and satisfaction for a finalized product. The BEST method is:
>
> A. Heuristic evaluation
> B. Summative user testing with quantitative metrics
> C. Cognitive walkthrough
> D. Pluralistic walkthrough

**Correct answer:** B — Measuring time and satisfaction requires real users (summative testing). Expert methods cannot measure these.

---

## Trap 4: Thinking Unmoderated Testing Is Inferior

**Trap:** Unmoderated testing is not inferior — it is different. It is faster, cheaper, and better for large samples, but lacks probing and adaptation.

**Example question:**
> A product team needs feedback from 100 users across 5 countries in one week. The MOST practical approach is:
>
> A. In-person moderated testing in each country
> B. Remote unmoderated testing
> C. Heuristic evaluation with 5 experts
> D. Pluralistic walkthroughs in each country

**Correct answer:** B — Remote unmoderated testing can scale to 100 users quickly and across countries.

---

## Trap 5: Thinking Guideline Reviews Are Only for Accessibility

**Trap:** Guideline reviews can evaluate against any standard: usability (ISO 9241), platform design (Apple HIG), or corporate standards. Not just accessibility.

**Example question:**
> A guideline review checks a mobile app against the Apple Human Interface Guidelines. This is an example of:
>
> A. Accessibility testing only
> B. A guideline-based review for platform consistency
> C. A cognitive walkthrough
> D. A user-based evaluation

**Correct answer:** B — Checking against platform guidelines is a guideline review for consistency, not specifically accessibility testing.
