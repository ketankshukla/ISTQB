# 📝 CT-SEC Mock Exam 3

**Instructions:** 40 questions, 60 minutes, single best answer.

### 🔷 Q1-Q10

1. The CIA triad includes: A) Authentication, Authorization, Auditing B) Confidentiality, Integrity, Availability C) Authentication, Integrity, Authorization D) Availability, Authorization, Auditing — **Confidentiality, Integrity, Availability**
2. Encryption of stored data primarily protects: A) Integrity B) Availability C) Confidentiality D) Authentication — **Confidentiality**
3. Verifying user identity is: A) Authorization B) Authentication C) Auditing D) Accounting — **Authentication**
4. Defense in depth means: A) One strong control B) Multiple layers of controls C) Complex design D) Secret design — **Multiple layers of controls**
5. Least privilege means: A) Admin access for all B) Minimum access necessary C) Invisible security D) Expensive controls — **Minimum access necessary**
6. Fail secure vs fail safe: A) Always same B) Secure = deny access, Safe = allow for physical safety C) Always opposite D) Never different — **Secure = deny access, Safe = allow for physical safety**
7. Complete mediation means: A) Check only at login B) Every access checked for authority C) Never challenge users D) Optional checks — **Every access checked for authority**
8. Economy of mechanism suggests: A) Maximum complexity B) Simpler = fewer vulnerabilities C) More features D) Expensive solutions — **Simpler = fewer vulnerabilities**
9. Open design principle: A) Keep design secret B) Security not dependent on secrecy C) Publish passwords D) Hide vulnerabilities — **Security not dependent on secrecy**
10. Psychological acceptability: A) Security at any usability cost B) Security should not impede usability C) Only for experts D) Ignore users — **Security should not impede usability**

### 🔷 Q11-Q20

11. Preventive control example: A) IDS alert B) Firewall blocking C) Backup restoration D) Audit log — **Firewall blocking**
12. Detective control example: A) Encryption B) IDS alert C) Account lockout D) Input validation — **IDS alert**
13. Corrective control example: A) WAF rule B) Intrusion detection C) Backup restoration D) Warning banner — **Backup restoration**
14. Security requirement vs functional: A) Same thing B) Security specifies HOW to protect C) Functional is security D) No difference — **Security specifies HOW to protect**
15. STRIDE: Spoofing violates: A) Integrity B) Authentication C) Confidentiality D) Availability — **Authentication**
16. STRIDE: Tampering violates: A) Authentication B) Integrity C) Confidentiality D) Availability — **Integrity**
17. STRIDE: Repudiation violates: A) Authentication B) Integrity C) Non-repudiation D) Availability — **Non-repudiation**
18. STRIDE: Information Disclosure violates: A) Authentication B) Integrity C) Confidentiality D) Availability — **Confidentiality**
19. STRIDE: Denial of Service violates: A) Authentication B) Integrity C) Confidentiality D) Availability — **Availability**
20. STRIDE: Elevation of Privilege violates: A) Authentication B) Authorization C) Confidentiality D) Availability — **Authorization**

### 🔷 Q21-Q30

21. Threat modeling purpose: A) Replace testing B) Identify threats early C) Remove all threats D) Increase complexity — **Identify threats early**
22. DREAD: Damage measures: A) Reproducibility B) How much damage C) Ease of exploit D) Users affected — **How much damage**
23. DREAD: Discoverability measures: A) How easy to find B) How much damage C) Ease of exploit D) Users affected — **How easy to find**
24. DREAD score calculation: A) Sum of all factors B) Average of 5 factors C) Highest factor D) Lowest factor — **Average of 5 factors**
25. Attack tree OR node: A) All children required B) Any child achieves goal C) Two children needed D) No children valid — **Any child achieves goal**
26. Attack tree AND node: A) Any child achieves goal B) All children required C) Two children needed D) No children valid — **All children required**
27. Risk formula: A) Threat + Vulnerability B) Likelihood x Impact C) Control - Asset D) Attack / Defense — **Likelihood x Impact**
28. Risk treatment: Mitigate means: A) Ignore B) Implement controls to reduce risk C) Transfer to insurance D) Accept — **Implement controls to reduce risk**
29. Risk treatment: Transfer means: A) Ignore B) Fix immediately C) Shift to another party D) Accept — **Shift to another party**
30. Risk treatment: Accept means: A) Never acceptable B) Acknowledge and document low risk C) Fix immediately D) Transfer — **Acknowledge and document low risk**

### 🔷 Q31-Q40

31. Penetration testing: Black box means: A) Full knowledge B) No knowledge C) Partial knowledge D) Source code access — **No knowledge**
32. Penetration testing: White box means: A) No knowledge B) Full knowledge C) Partial knowledge D) Only network access — **Full knowledge**
33. Penetration testing: Gray box means: A) No knowledge B) Full knowledge C) Partial knowledge D) No access — **Partial knowledge**
34. Penetration phase: Reconnaissance is: A) Exploitation B) Information gathering C) Reporting D) Cleanup — **Information gathering**
35. Penetration phase: Exploitation is: A) Information gathering B) Actually exploiting vulnerabilities C) Reporting D) Planning — **Actually exploiting vulnerabilities**
36. Horizontal privilege escalation: A) Higher privileges B) Same level, different user C) Admin access D) Root access — **Same level, different user**
37. Vertical privilege escalation: A) Same level B) Higher privileges C) Different user D) No change — **Higher privileges**
38. IDOR stands for: A) International Data Object Reference B) Insecure Direct Object Reference C) Internal Document Object Reference D) Invalid Data Output Reference — **Insecure Direct Object Reference**
39. Session fixation: A) Good practice B) Pre-auth session ID persists after login C) Session changes D) Token rotation — **Pre-auth session ID persists after login**
40. Session hijacking: A) Stealing session identifier B) Creating new session C) Logging out D) Changing password — **Stealing session identifier**
