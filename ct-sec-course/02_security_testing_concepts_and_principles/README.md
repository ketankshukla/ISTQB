# 📜 Chapter 2 — Security Testing Concepts and Principles

## 🎓 Exam Weighting

- **Estimated weight:** ~15% (~6 questions out of 40)
- **K-levels:** K2 (understand), K3 (apply)
- **Focus:** Security principles, controls, requirements, secure development

---

## 🎯 Learning Objectives

By the end of this chapter, you should be able to:

1. Explain key security principles (defense in depth, least privilege, fail secure)
2. Describe security controls and their types
3. Differentiate security requirements from functional requirements
4. Describe the role of security in the SDLC
5. Apply security principles to testing scenarios

---

## ⭐ Key Concepts

- Defense in depth
- Least privilege, separation of duties
- Fail secure / fail safe
- Complete mediation, economy of mechanism
- Open design, psychological acceptability
- Security controls (preventive, detective, corrective)
- Security requirements engineering
- Secure SDLC
