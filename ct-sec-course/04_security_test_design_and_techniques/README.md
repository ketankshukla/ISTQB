# Chapter 4 — Security Test Design and Techniques

## Exam Weighting

- **Estimated weight:** ~20% (~8 questions out of 40)
- **K-levels:** K2 (understand), K3 (apply)
- **Focus:** Penetration testing, fuzzing, static/dynamic analysis, input validation, authentication testing

---

## Learning Objectives

By the end of this chapter, you should be able to:

1. Describe penetration testing phases and types
2. Design tests for authentication and authorization
3. Apply input validation and output encoding tests
4. Describe fuzzing techniques
5. Differentiate static and dynamic analysis
6. Design tests for session management and cryptography

---

## Key Concepts

- Penetration testing (black box, gray box, white box)
- Reconnaissance, scanning, exploitation, post-exploitation, reporting
- Authentication testing (brute force, credential stuffing, MFA bypass)
- Authorization testing (horizontal/vertical privilege escalation)
- Input validation and output encoding
- Fuzzing (mutation, generation-based)
- SAST, DAST, IAST
- Session management testing
- Cryptographic testing
