# CT-SEC Exam Strategy

## Exam Format

| Parameter | Value |
|-----------|-------|
| Total questions | 40 |
| Duration | 60 minutes |
| Time per question | 1.5 minutes |
| Pass mark | 65% (26/40) |
| Target score | 32+/40 (80%+) |

---

## Time Budget

| Phase | Time | Action |
|-------|------|--------|
| Quick pass | 45 min | Answer all easy/medium questions |
| Flagged review | 10 min | Return to flagged questions |
| Final check | 5 min | Verify no blanks, review uncertain answers |

**If stuck on a question:** Mark it, move on, return later. Never spend more than 2.5 minutes on one question.

---

## Chapter Weighting and Tactics

| Chapter | Weight | K-Levels | Tactics |
|---------|--------|----------|---------|
| 1 — Introduction | ~15% | K1-K2 | Memorize CIA triad, security terms, OWASP categories |
| 2 — Concepts and Principles | ~15% | K2-K3 | Understand defense in depth, least privilege, fail secure |
| 3 — Threat Modeling and Risk | ~20% | K2-K3 | Master STRIDE, DREAD, risk calculation, attack trees |
| 4 — Test Design and Techniques | ~20% | K2-K3 | Know penetration testing phases, fuzzing, static/dynamic analysis |
| 5 — Test Execution and Evaluation | ~15% | K2-K3 | Understand vulnerability scoring (CVSS), test reporting |
| 6 — Tools and Automation | ~15% | K2 | Recognize SAST, DAST, IAST, SCA tools |

---

## K-Level Patterns

**K1 (Remember):** Direct recall of definitions
- Example: "The three pillars of information security are Confidentiality, Integrity, and..."
- Strategy: Memorize glossary terms precisely

**K2 (Understand):** Comprehend concepts and explain
- Example: "Why is defense in depth more effective than a single security control?"
- Strategy: Focus on "why" and "how" in lessons

**K3 (Apply):** Use knowledge in a scenario
- Example: "Given a web application login flow, identify which STRIDE threats apply."
- Strategy: Practice case studies and worked examples

---

## Common Distractor Patterns

1. **Confusing security with functional testing**
   - Trap: Thinking that functional tests verify security
   - Reality: Security tests target specific vulnerabilities, not just features

2. **Confusing SAST, DAST, IAST, and SCA**
   - Trap: Mixing up when each is used and what they detect
   - Reality: Each has distinct use cases (source code vs. running app vs. libraries)

3. **Overlooking defense in depth**
   - Trap: Thinking one strong control is sufficient
   - Reality: Multiple layers of controls provide better protection

4. **Confusing CVSS components**
   - Trap: Mixing up base, temporal, and environmental scores
   - Reality: Each component has specific metrics and purposes

5. **Thinking security testing is only penetration testing**
   - Trap: Equating all security testing with pen testing
   - Reality: Security testing includes reviews, static analysis, risk assessment, and more

## Pre-Exam Checklist

- [ ] All 6 chapters completed with practice scores 75%+
- [ ] All 3 case studies completed with model answers reviewed
- [ ] All 3 mock exams completed with scores 28+/40 improving to 32+/40
- [ ] Glossary terms reviewed twice
- [ ] Exam traps from all chapters reviewed
- [ ] Weak areas identified and re-studied

---

## During the Exam

1. **Read the full question** before looking at answers
2. **Eliminate obviously wrong answers** first
3. **Look for key phrases** that distinguish similar concepts
4. **For scenario questions:** Identify the core issue before selecting
5. **Don't overthink** — first instinct is often correct for K1/K2
6. **Mark uncertain questions** and return with fresh perspective

## Post-Exam

- Note any unexpected topics for future learners
- Celebrate — you've prepared thoroughly!
