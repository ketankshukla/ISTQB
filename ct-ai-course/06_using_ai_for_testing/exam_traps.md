# 🪤 Chapter 6 — Using AI for Testing: Exam Traps

## 🪤 Trap 1: Thinking AI Can Fully Replace Human Testers

**Trap:** AI testing tools are assistants, not replacements. Humans are needed for strategy, complex scenarios, and ethical judgment.

**Example question:**
> A company replaces all human testers with an AI testing tool. The risk is:
>
> A. The AI will test faster and more thoroughly than humans
> B. AI cannot understand complex business context, design comprehensive strategies, or exercise human judgment
> C. There is no risk — AI is superior
> D. The AI will find fewer defects

**Correct answer:** B — AI augments human testing but cannot replace strategic thinking, creativity, and business understanding.

---

## 🪤 Trap 2: Trusting AI-Generated Tests Without Validation

**Trap:** AI-generated tests may be incorrect, irrelevant, or miss important scenarios. Human review is essential.

**Example question:**
> An AI tool generates 500 test cases. The team runs all of them without review. This is:
>
> A. Best practice — AI-generated tests are always correct
> B. Risky — AI may generate invalid or irrelevant tests that waste time or provide false confidence
> C. Faster than manual test creation
> D. Only appropriate for unit tests

**Correct answer:** B — AI-generated tests must be validated by humans before execution.

---

## 🪤 Trap 3: Thinking All AI Testing Tools Are the Same

**Trap:** AI testing tools vary in capability, accuracy, and suitability. Selection must match the specific testing need.

**Example question:**
> A team selects an AI testing tool because it has the most marketing. The tool is designed for web visual testing, but the team needs API testing. The problem is:
>
> A. The tool will work for API testing anyway
> B. The tool does not fit the team's specific testing need
> C. Marketing is the best selection criterion
> D. All AI tools are equivalent

**Correct answer:** B — Tool selection must match the specific testing need.

---

## 🪤 Trap 4: Ignoring Data Privacy with AI Testing Tools

**Trap:** AI testing tools may send code, test data, or screenshots to cloud services for analysis.

**Example question:**
> A healthcare company uses a cloud-based AI test generation tool that uploads their application code for analysis. The risk is:
>
> A. No risk — cloud tools are always secure
> B. Patient data and proprietary code may be exposed to third parties
> C. The tool will generate better tests
> D. Cloud tools are faster

**Correct answer:** B — Cloud-based AI tools may expose sensitive data. Data privacy must be evaluated.

---

## 🪤 Trap 5: Thinking Autonomous Testing Is Fully Autonomous Today

**Trap:** True Level 5 autonomous testing does not exist. Current tools require human setup, scope definition, and result validation.

**Example question:**
> An "autonomous testing" product claims it will eliminate the need for any human testing involvement. This claim is:
>
> A. True — autonomous means fully independent
> B. False — current autonomous testing still requires human oversight, scope definition, and validation
> C. True only for simple applications
> D. True if the tool uses deep learning

**Correct answer:** B — Current autonomous testing tools still require significant human involvement.
