# 📖 CT-AI Glossary

Key terms and definitions for the CT-AI exam, organized by chapter relevance.

---

## 🏗️ AI Fundamentals

**Artificial Intelligence (AI)**
The simulation of human intelligence processes by computer systems, including learning, reasoning, and self-correction.

**Machine Learning (ML)**
A subset of AI where systems learn from data without being explicitly programmed.

**Deep Learning**
A subset of machine learning based on artificial neural networks with multiple layers.

**Neural Network**
A computing system inspired by biological neural networks with input, hidden, and output layers.

**Supervised Learning**
ML where the model is trained on labeled data (input-output pairs).

**Unsupervised Learning**
ML where the model finds patterns in unlabeled data.

**Reinforcement Learning**
ML where an agent learns by interacting with an environment and receiving rewards or penalties.

**Classification**
A supervised learning task where the output is a discrete category.

**Regression**
A supervised learning task where the output is a continuous value.

**Clustering**
An unsupervised learning task where data is grouped into clusters.

**Feature**
An individual measurable property of the data.

**Label**
The output value or category in supervised learning.

**Training Data**
The dataset used to train a machine learning model.

**Test Data**
The dataset used to evaluate a trained model. Must be separate from training data.

**Validation Data**
A dataset used during training to tune hyperparameters.

**Model**
The learned representation produced by an ML algorithm.

**Inference**
Using a trained model to make predictions on new data.

**Natural Language Processing (NLP)**
AI techniques for understanding and generating human language.

**Computer Vision**
AI techniques for interpreting visual information.

**Expert System**
An AI system using a knowledge base and inference rules.

**Robotic Process Automation (RPA)**
Software robots that automate repetitive, rule-based tasks.

## 📐 ML Model Testing

**Accuracy**
The proportion of total predictions that were correct.

**Precision**
The proportion of positive predictions that were actually correct.

**Recall**
The proportion of actual positives that were correctly identified.

**F1-Score**
The harmonic mean of precision and recall.

**Confusion Matrix**
A table showing TP, TN, FP, FN counts.

**True Positive (TP)**
Correctly predicted positive class.

**True Negative (TN)**
Correctly predicted negative class.

**False Positive (FP)**
Incorrectly predicted positive class (Type I error).

**False Negative (FN)**
Incorrectly predicted negative class (Type II error).

**AUC-ROC**
Area Under the Receiver Operating Characteristic curve.

**Overfitting**
When a model learns the training data too well, including noise, and performs poorly on new data.

**Underfitting**
When a model is too simple to capture the underlying pattern in the data.

**Cross-Validation**
Technique where data is split into k folds; model is trained on k-1 folds and validated on the remaining fold.

**Bias (in ML)**
Systematic error causing the model to consistently deviate from the true value.

**Variance**
The amount the model's predictions change for different training data.

**Bias-Variance Tradeoff**
The balance between underfitting (high bias) and overfitting (high variance).

**Hyperparameter**
Parameters set before training that control the learning process (e.g., learning rate, number of layers).

## 🤖 AI Testing Concepts

**Test Oracle**
A mechanism for determining whether a test has passed or failed. For AI systems, oracles are often probabilistic or statistical.

**Metamorphic Testing**
A testing technique where relationships between inputs and outputs are checked rather than specific outputs.

**A/B Testing**
Comparing two versions of a model or system by randomly assigning users to each version.

**Non-Determinism**
The property where the same input may produce different outputs due to randomness in training or inference.

**Adversarial Example**
An input deliberately designed to cause a model to make a mistake.

**Data Drift**
When the statistical properties of input data change over time, degrading model performance.

**Concept Drift**
When the relationship between inputs and outputs changes over time.

**Model Drift**
Degradation of model performance over time due to changing data or environment.

**Fairness**
The principle that AI systems should not discriminate against individuals or groups based on protected characteristics.

**Explainability**
The ability to understand and interpret why an AI system made a particular decision.

**Robustness**
The ability of an AI system to maintain performance under varying conditions, including unexpected or adversarial inputs.

**Ethics in AI**
Principles ensuring AI systems are transparent, accountable, and beneficial to society.

---

## 🤖 AI for Testing

**Autonomous Testing**
Testing where AI independently generates, executes, and evaluates tests without human intervention.

**Test Generation with AI**
Using AI to automatically create test cases from requirements, code, or user behavior data.

**Defect Prediction**
Using AI to predict which parts of code are likely to contain defects.

**Visual Testing**
Using computer vision to detect UI regressions by comparing screenshots.

**Intelligent Test Prioritization**
Using AI to order tests by likelihood of finding defects or business risk.
