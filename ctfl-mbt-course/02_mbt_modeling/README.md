# Chapter 2 — MBT Modeling

## Exam Weighting

- **Estimated weight:** ~25% (~10 questions out of 40)
- **K-levels:** K2 (understand), K3 (apply)
- **Focus:** State machines, decision tables, deriving models from requirements

---

## Learning Objectives

By the end of this chapter, you should be able to:

1. Describe state machines and their components (states, transitions, events, guards, actions)
2. Describe decision tables and their components (conditions, actions, rules)
3. Explain how to derive models from requirements
4. Identify appropriate model types for different specifications
5. Describe the relationship between models and the system under test

---

## Key Concepts

- State machine components: states, transitions, events, guard conditions, actions
- Decision table components: conditions, actions, rules, impossible rules
- Deriving models from requirements, user stories, and use cases
- Model validation: ensuring the model reflects the SUT
- Common modeling notations
