# 🪤 Chapter 5 — Tools and Automation: Exam Traps

## 🪤 Trap 1: Thinking MBT Tools Eliminate the Need for Models

**Trap:** Tools generate tests FROM models. Models are still required.

**Example question:**
> An MBT test generator tool:
>
> A. Creates models automatically from requirements
> B. Generates test cases from models created by testers
> C. Eliminates the need for any human input
> D. Replaces the system under test

**Correct answer:** B — Tools generate tests from models, not from thin air.

---

## 🪤 Trap 2: Thinking One Tool Does Everything

**Trap:** Most teams use a combination of tools: model editor + test generator + execution framework.

**Example question:**
> A team needs to create state machine models, generate tests, and execute them in CI. The BEST approach is:
>
> A. Find a single tool that does all three perfectly
> B. Select best-of-breed tools for each need and integrate them
> C. Use only open-source tools regardless of fit
> D. Build all tools from scratch

**Correct answer:** B — Best-of-breed tools integrated together is typically the best approach.

---

## 🪤 Trap 3: Ignoring Tool Learning Curve

**Trap:** MBT tools require modeling knowledge and tool expertise. Training is essential.

**Example question:**
> A team purchases an expensive MBT tool but sees no improvement in testing after 3 months. What is the MOST likely cause?
>
> A. The tool is defective
> B. The team lacks training in modeling and the tool
> C. The system is too simple for MBT
> D. The tool is too cheap

**Correct answer:** B — Training and modeling expertise are essential for MBT success.

---

## 🪤 Trap 4: Thinking Manual MBT Is Never Appropriate

**Trap:** For small projects or pilots, manual modeling and test generation may be sufficient.

**Example question:**
> A 2-person team is testing a simple system with 3 states. They should:
>
> A. Purchase an enterprise MBT tool immediately
> B. Use manual modeling and test generation for now
> C. Skip modeling entirely
> D. Hire 5 more testers

**Correct answer:** B — For small/simple systems, manual MBT may be more cost-effective.
