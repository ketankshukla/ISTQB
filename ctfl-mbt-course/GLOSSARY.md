# 📖 CTFL-MBT Glossary

---

## ⭐ A

**Abstract Test Case**
A test case derived from a model that may need further refinement or concretization before it can be executed.

**Action**
A behavior or operation that occurs when a transition is taken in a state machine.

---

## ⭐ B

**Boundary Testing**
Testing at the boundaries of partitions derived from a model.

---

## ⭐ C

**Concrete Test Case**
An executable test case with specific input values and expected results.

**Coverage Criterion**
A rule or condition that defines which elements of a model must be exercised by generated test cases.

---

## ⭐ D

**Decision Table**
A tabular representation of conditions and actions used to model business rules and generate test cases.

**Defect Detection Rate**
The number of defects found per unit of testing effort.

---

## ⭐ E

**Event**
A trigger that causes a transition from one state to another in a state machine.

**Executable Test Case**
A test case that can be run against the system under test.

---

## ⭐ F

**Full State Coverage**
A coverage criterion requiring that every state in a state machine is visited at least once.

---

## ⭐ G

**Guard Condition**
A Boolean condition that must be true for a transition to be enabled.

---

## ⭐ I

**Infeasible Path**
A path through a model that cannot be executed in the real system due to implementation constraints.

---

## ⭐ M

**Model**
An abstract representation of the behavior or structure of a system under test.

**Model-Based Testing (MBT)**
A testing approach where test cases are generated from models of the system under test.

**Model Coverage**
The degree to which elements of a model (states, transitions, paths) are exercised by test cases.

---

## ⭐ P

**Path**
A sequence of transitions through a state machine from an initial state to a final state.

**Path Coverage**
A coverage criterion requiring that specific paths through a model are exercised.

---

## ⭐ S

**Selection Criterion**
A rule used to determine which test cases to generate from a model.

**State**
A condition or mode of a system in a state machine.

**State Machine**
A model representing system behavior as states, transitions, events, and actions.

**State Transition Testing**
A test design technique using state machine models to derive test cases.

**Switch Coverage**
A family of coverage criteria based on sequences of transitions (0-switch, 1-switch, 2-switch).

---

## ⭐ T

**Test Case Generation**
The process of deriving test cases from a model using selection criteria.

**Test Oracle**
A source of expected results for generated test cases.

**Transition**
A change from one state to another, triggered by an event.

**Transition Coverage**
A coverage criterion requiring that every transition in a state machine is exercised at least once.

**Transition Pair Coverage**
A coverage criterion requiring that every pair of consecutive transitions is exercised.

---

## ⭐ U

**Unreachable State**
A state in a model that cannot be reached from the initial state through any sequence of transitions.
