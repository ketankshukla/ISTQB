# Chapter 4 — MBT Test Execution and Analysis

## Exam Weighting

- **Estimated weight:** ~15% (~6 questions out of 40)
- **K-levels:** K2 (understand)
- **Focus:** Test execution, coverage measurement, defect analysis, test reporting

---

## Learning Objectives

By the end of this chapter, you should be able to:

1. Describe how abstract test cases are made executable
2. Explain coverage measurement in MBT
3. Describe defect analysis from an MBT perspective
4. Explain test reporting for MBT
5. Describe how to handle infeasible paths and unreachable states

## Key Concepts

- Concretization of abstract test cases
- Coverage measurement and reporting
- Defect classification in MBT
- Handling infeasible paths
- Test execution in MBT
