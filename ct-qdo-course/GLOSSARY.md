# 📖 CT-QDO Glossary

## ⭐ A

- **Artifact Repository** — Storage for build artifacts (e.g., Nexus, Artifactory)
- **A/B Testing** — Comparing two versions of software in production to determine which performs better

## ⭐ C

- **Canary Deployment** — Deploying to a small subset of users before full rollout
- **CI/CD** — Continuous Integration / Continuous Delivery (or Deployment)
- **Configuration Drift** — Difference between intended and actual environment configurations
- **Container** — Lightweight isolated runtime environment (e.g., Docker)

## ⭐ D

- **DevOps** — Culture and practice of collaboration between development and operations
- **Dark Launch** — Deploying features to production but keeping them invisible to users

## ⭐ E

- **Environment Parity** — Keeping dev, test, and production environments as similar as possible

## ⭐ G

- **GitOps** — Using Git as the single source of truth for declarative infrastructure and applications
- **Golden Image** — Pre-configured, tested, and approved base environment image

## ⭐ I

- **IaC (Infrastructure as Code)** — Managing infrastructure through code and automation
- **Immutable Infrastructure** — Servers are never modified after deployment; replaced instead

## ⭐ L

- **Log Aggregation** — Collecting logs from multiple services into a central system

## ⭐ M

- **Mean Time To Detect (MTTD)** — Average time to detect an issue
- **Mean Time To Repair (MTTR)** — Average time to fix an issue
- **Microservices** — Architectural approach where application is composed of small, independent services

## ⭐ O

- **Observability** — Ability to understand system state from outputs (logs, metrics, traces)
- **Orchestration** — Automating deployment, scaling, and management of containers (e.g., Kubernetes)

## ⭐ P

- **Pipeline as Code** — Defining CI/CD pipelines in version-controlled code
- **Production Telemetry** — Real-time data collected from production systems

## ⭐ Q

- **Quality Gate** — Checkpoint in pipeline that blocks progression if criteria not met

## ⭐ S

- **Service Mesh** — Infrastructure layer for service-to-service communication (e.g., Istio)
- **Shift-Left** — Moving testing activities earlier in the development lifecycle
- **Shift-Right** — Testing and monitoring in production environments
- **Smoke Test** — Quick verification that basic functionality works after deployment
- **SRE (Site Reliability Engineering)** — Discipline applying software engineering to operations

## ⭐ T

- **Telemetry** — Automated collection and transmission of system data
- **Test Environment Parity** — Keeping test environments consistent with production
- **Trace** — End-to-end record of a request as it flows through distributed services

## ⭐ V

- **Value Stream Mapping** — Visualizing the flow of value delivery from idea to production
