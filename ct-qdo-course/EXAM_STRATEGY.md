# CT-QDO Exam Strategy

## Exam Format

- 40 questions in 60 minutes
- Closed book
- Single best answer (some "choose N")
- 65% pass mark (26/40 correct)

## Time Budget

- 60 minutes / 40 questions = 1.5 minutes per question
- Flag uncertain questions, return at the end
- Reserve 5 minutes for review

## K-Level Patterns

- **K1 (Remember):** DevOps principles, CI/CD stages, observability terms
- **K2 (Understand):** How shift-left reduces defects, why monitoring matters, pipeline gates
- **K3 (Apply):** Design a pipeline test strategy, select monitoring metrics, configure quality gates

## Common Distractor Patterns

- **"Shift-left" vs "Shift-right"** — Shift-left = early testing. Shift-right = production monitoring.
- **"Continuous Integration" vs "Continuous Delivery"** — CI = build and test. CD = deploy to production.
- **"Unit test" vs "Integration test" in pipeline** — Unit fast, integration slower. Run unit first.

## Pre-Exam Checklist

- [ ] All mock exams completed with 80%+
- [ ] Glossary terms memorized
- [ ] Exam traps reviewed
- [ ] Case studies completed

## Exam Day Advice

- Read each question fully before answering
- Watch for absolutes ("always," "never") — they are usually wrong
- For "choose N" questions, verify you selected exactly N
