# ❓ Chapter 3 — Shift-Left Testing Practices

- **Estimated weight:** ~20% (~8 questions out of 40)
- **K-levels:** K2, K3

## 🎯 Learning Objectives

By the end of this chapter, you should be able to:

1. Explain shift-left testing concepts
2. Describe TDD and BDD in DevOps
3. Implement contract testing for microservices
4. Use static analysis and linters in pipelines
5. Integrate code review and pair programming
6. Design tests that run before deployment

- Shift-left vs traditional testing

- Static analysis and linters
- Code review practices
