### 🔷 Q1-Q10

1. DevOps is primarily: A) A set of tools B) A cultural movement emphasizing collaboration C) A replacement for agile D) Only faster deployments — **A cultural movement emphasizing collaboration**
2. The "M" in CALMS: A) Management B) Measurement C) Monitoring D) Methodology — **Measurement**
3. First Way (Flow): A) Production monitoring only B) Optimizing the entire value stream C) Only post-mortems D) Only automated testing — **Optimizing the entire value stream**
4. Quality in DevOps: A) Only QA team B) Everyone's responsibility C) Only before release D) Not important — **Everyone's responsibility**
5. Blameless post-mortem: A) Identifies who caused it B) Focuses on system improvements C) Assigns penalties D) Ignores incidents — **Focuses on system improvements**
6. Value stream mapping: A) Track vacations B) Identify bottlenecks and waste C) Only CPU monitoring D) Only budgets — **Identify bottlenecks and waste**
7. Shift-left: A) Only production testing B) Testing earlier in lifecycle C) Delaying testing D) Only maintenance — **Testing earlier in lifecycle**
8. CI typically: A) Manual reviews only B) Automated build and test on every commit C) Production without testing D) Only monthly — **Automated build and test on every commit**
9. Second Way (Feedback): A) Only deployments B) Amplifying feedback loops C) Eliminating testing D) Increasing team size — **Amplifying feedback loops**
10. Small batches: A) Require less documentation B) Reduce risk and enable faster feedback C) Eliminate testing D) Required by regulators — **Reduce risk and enable faster feedback**
