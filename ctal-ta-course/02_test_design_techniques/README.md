## Learning Objectives

By the end of this chapter, you should be able to:

1. Apply equivalence partitioning to derive test cases from specifications
2. Apply boundary value analysis (2-value and 3-value) to identify test cases
3. Apply decision table testing to complex business rules
4. Apply state transition testing to state-based systems
5. Apply use case testing to workflow-based specifications
6. Understand combinatorial testing (pairwise) for multiple parameters
7. Describe error guessing and when it is appropriate
8. Describe checklist-based testing
9. Describe exploratory testing, test charters, and session-based test management
10. Select the most appropriate technique(s) for a given specification and risk profile

---

## Key Concepts to Master

- Equivalence partitioning: valid and invalid partitions, how to count test cases
- Boundary value analysis: min-1, min, min+, max-, max, max+1
- Decision tables: conditions, actions, rules, impossible rules, redundant rules
- State transitions: states, events, transitions, actions, 0-switch and 1-switch coverage
- Use cases: actors, preconditions, main path, alternatives, exceptions
- Combinatorial testing: pairwise, all-pairs, orthogonal arrays
- Error guessing: experience-driven, not systematic, valuable supplement
- Exploratory testing: simultaneous learning, design, and execution; test charters

## Exam Tips

- Expect K3/K4 questions that require you to APPLY techniques to specifications
- You must be able to count test cases for EP, BVA, and decision tables
- State transition questions often ask for valid vs. invalid transitions
- Technique selection questions require matching the specification structure to the technique
- Practice with scratch paper — technique application questions require calculations
