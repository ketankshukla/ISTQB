# ISTQB CTAL-TA v4.0 — Advanced Test Analyst Self-Study Course

Complete self-study course for the **ISTQB Certified Tester Advanced Level — Test Analyst (CTAL-TA)** examination.

---

## Exam Facts

| Parameter | Value |
|-----------|-------|
| Full name | ISTQB Certified Tester Advanced Level — Test Analyst |
| Syllabus version | v4.0 |
| Total questions | 60 |
| Duration | 180 minutes (3 hours) |
| Pass mark | 65% |
| Question types | Multiple choice (single best answer, scenario-based) |
| Materials allowed | None (closed book) |
| Negative marking | No |
| Prerequisites | ISTQB CTFL v4.0 (or compatible Foundation Level) |

---

## Course Structure

This course is organized into 5 chapters aligned with the CTAL-TA v4.0 syllabus:

| Chapter | Topic | Weight | K-Levels |
|---------|-------|--------|----------|
| 1 | Testing Process | ~15% | K2, K3 |
| 2 | Test Design Techniques | ~30% | K2, K3, K4 |
| 3 | Testing Software Quality Characteristics | ~25% | K2, K3 |
| 4 | Reviews | ~10% | K2, K3 |
| 5 | Test Tools and Automation | ~20% | K2, K3 |

**Key insight:** Chapter 2 (Test Design Techniques) is the highest-weighted and most technically demanding chapter, requiring you to **apply** techniques to scenarios (K3/K4). Master this chapter and you have a strong floor.

---

## How to Use This Course

1. Follow the `STUDY_PLAN.md` — it's designed for 4-6 weeks of study at 8-12 hours/week.
2. Read each chapter's `lessons.md` thoroughly — this is your primary learning material.
3. Review the chapter `README.md` for exam weighting and learning objectives.
4. Study `exam_traps.md` to avoid common mistakes.
5. Complete all `practice_questions.md` — don't just read them; time yourself.
6. Work through all 3 case studies — they integrate multiple chapters.
7. Attempt the mock exams under timed conditions.
8. Review `EXAM_STRATEGY.md` in the final week.

## Prerequisites

- **Mandatory:** ISTQB CTFL v4.0 (Certified Tester Foundation Level)
- Recommended: 2+ years of hands-on testing experience
- Recommended: Familiarity with test design techniques (equivalence partitioning, boundary value analysis)
- Recommended: Experience with at least one test management tool and one defect tracking tool

- **Aligned with syllabus:** All content maps to CTAL-TA v4.0 learning objectives.
- **K-level appropriate:** K1 content is presented for recall; K2 for understanding; K3/K4 for application.
- **Scenario-driven:** K3/K4 questions require applying techniques to realistic testing scenarios.

## Certification Context

The CTAL-TA is part of the ISTQB Advanced Level certification track. After achieving CTAL-TA, you may consider:

- **CTAL-TM** (Test Manager) — for test leadership roles
- **CTAL-TTA** (Technical Test Analyst) — for white-box and technical testing
- **CT-TAE** (Test Automation Engineering) — for automation specialization
- **CT-SEC** (Security Testing) — for security-focused testing
- **CT-PT** (Performance Testing) — for load and performance testing

For a complete certification path map, see `ISTQB_Certification_Paths.md` in the repository root.
