## Learning Objectives

By the end of this chapter, you should be able to:

1. Describe how test activities fit into software development lifecycles (sequential, iterative, incremental, agile)
2. Explain the purpose and content of test planning, monitoring, and control activities
3. Identify testable requirements and derive test conditions during test analysis
4. Apply test design techniques to create test cases from test conditions
5. Describe test implementation activities (test environment setup, test data preparation)
6. Explain test execution entry criteria, suspension criteria, and resumption criteria
7. Describe the process of evaluating exit criteria and reporting test results
8. List test closure activities and their purpose

---

## Key Concepts to Master

- Test activities and their sequencing in different lifecycle models
- Test planning content (test strategy, test plan, estimates, schedules)
- Test monitoring metrics (test coverage, defect density, test progress)
- Test analysis: identifying what to test (test conditions, test charters)
- Test design: designing how to test (test cases, test procedures)
- The difference between test analysis and test design
- Test implementation: preparing to run tests
- Test execution: running tests and recording results
- Exit criteria: conditions for stopping testing
- Test closure: archiving, lessons learned, release readiness

---

## Exam Tips

- Expect scenario-based questions asking you to identify the appropriate test activity for a given situation
- Questions often test whether you understand the difference between test analysis and test design
- Know what goes into a test plan vs. what is decided during test monitoring
- Understand the relationship between test activities and development activities in different lifecycle models
