# 🪤 Chapter 5 — Platform and Compliance Testing: Exam Traps

## 🪤 Trap 1: Certification is Just a Formality

**Trap:** Certification has strict requirements; failure means delay or rejection.

**Example question:**
> Platform certification is a quick rubber-stamp process. This is:
>
> A. Correct — platform holders trust developers
> B. Incorrect — certification has detailed technical requirements; failure can block release
> C. Only for indie developers
> D. Only for large publishers

**Correct answer:** B — Certification is rigorous.

---

## 🪤 Trap 2: Ratings Apply Only to Explicit Content

**Trap:** Ratings cover implied content, context, and intensity, not just explicit material.

**Example question:**
> A game has no visible blood, but intense violence with screams and implied death. The rating should consider:
>
> A. Only visible content — no blood means E rating
> B. The intensity, context, and implied violence, not just visible blood
> C. Only audio volume
> D. Only character age

**Correct answer:** B — Ratings consider intensity and context.

---

## 🪤 Trap 3: Online Services Testing is Optional

**Trap:** Online services (trophies, leaderboards, multiplayer) must be tested; they're part of cert.

**Example question:**
> A single-player game doesn't need online service testing. This is:
>
> A. Correct — no online = no testing needed
> B. Incorrect — trophies/achievements, cloud saves, and patches still require online service testing
> C. Only for console
> D. Only for PC

**Correct answer:** B — Even single-player has online requirements.

---

## 🪤 Trap 4: Store Monetization Doesn't Need Testing

**Trap:** Purchase flows must be tested for correctness, security, and platform compliance.

**Example question:**
> In-app purchase testing is handled by the platform store, not the game. This is:
>
> A. Correct — Apple/Google handle payments
> B. Incorrect — the game must test purchase flow, receipt validation, item delivery, and restore functionality
> C. Only for free games
> D. Only for premium games

**Correct answer:** B — Game must test purchase integration.

---

## 🪤 Trap 5: Day-One Patch Means No Need for Disc Quality

**Trap:** Disc/cartridge build must be stable; not all players have internet for day-one patch.

**Example question:**
> Since a day-one patch will fix bugs, the disc build doesn't need to be high quality. This is:
>
> A. Correct — the patch will fix everything
> B. Incorrect — many players lack internet for patches; the disc build must be stable and playable on its own
> C. Only for console
> D. Only for physical copies

**Correct answer:** B — Disc build must be stable standalone.
