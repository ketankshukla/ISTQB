# Chapter 2 — Game Development Lifecycle: Exam Traps

## Trap 1: Alpha Means Feature Complete and Bug Free

**Trap:** Alpha = feature complete, but with known bugs. Beta = content complete, focusing on fixes.

**Example question:**
> Alpha milestone means the game is ready for release with no bugs. This is:
>
> A. Correct — alpha is release-ready
> B. Incorrect — alpha means core features are implemented but bugs are expected; gold is release-ready
> C. Only for AAA games
> D. Only for mobile games

**Correct answer:** B — Alpha ≠ bug-free.

---

## Trap 2: Testing Only Happens at the End

**Trap:** Testing must start early (prototype evaluation, build verification) to catch issues cheaply.

**Example question:**
> Testing in game development should begin only after the beta milestone. This is:
>
> A. Correct — that's when most content exists
> B. Incorrect — testing starts in pre-production (prototype evaluation, test planning) and continues throughout; late testing is expensive
> C. Only for small teams
> D. Only for large teams

**Correct answer:** B — Test early and continuously.

---

## Trap 3: Live Service Games Don't Need Post-Launch Testing

**Trap:** Live service games require ongoing testing for each update, balance change, and event.

**Example question:**
> A live service game requires no testing after initial launch. This is:
>
> A. Correct — the launch build is final
> B. Incorrect — live games have continuous updates, events, and balance changes that each require testing
> C. Only for single-player games
> D. Only for offline games

**Correct answer:** B — Live service = continuous testing.

---

## Trap 4: Vertical Slice is Only for Marketing

**Trap:** Vertical slice validates production pipeline, team capability, and final quality target.

**Example question:**
> A vertical slice is only used for marketing trailers. This is:
>
> A. Correct — it's a demo for players
> B. Incorrect — it also validates production pipeline, team capability, and serves as a quality target for the full game
> C. Only for indie games
> D. Only for VR games

**Correct answer:** B — Vertical slice = pipeline validation + quality target.

---

## Trap 5: BVT is the Same as Full Regression

**Trap:** BVT is a quick sanity check (minutes), not a full test pass (hours/days).

**Example question:**
> A team runs their full 8-hour test suite as a build verification test. The problem is:
>
> A. No problem — thoroughness is good
> B. BVT should be a quick smoke test (minutes); running the full suite delays feedback and defeats the purpose
> C. Only that it's expensive
> D. Only that testers get tired

**Correct answer:** B — BVT = quick smoke, not full regression.
