# ♟️ CT-GaMe Exam Strategy

## 🎓 Exam Format

- 40 questions in 60 minutes
- Closed book
- Single best answer (some "choose N")
- 65% pass mark (26/40 correct)

## ⭐ Time Budget

- 60 minutes / 40 questions = 1.5 minutes per question
- Flag uncertain questions, return at the end
- Reserve 5 minutes for review

## ⭐ Chapter Weighting

| Chapter | Est. Questions | Priority |
|---------|---------------|----------|
| 2 — Game Development Lifecycle | ~8 | High |
| 3 — Functional Game Testing | ~8 | High |
| 5 — Platform and Compliance Testing | ~8 | High |
| 4 — Non-Functional Game Testing | ~6 | Medium |
| 6 — Test Automation and Tools | ~6 | Medium |
| 1 — Introduction | ~4 | Low |

## 🪜 K-Level Patterns

- **K1 (Remember):** Game genres, platform types, rating systems, compliance requirements
- **K2 (Understand):** Why exploratory testing is used; how compliance affects release
- **K3 (Apply):** Design test cases for a game feature; select platform test strategy

## ⭐ Common Distractor Patterns

- **"Game testing is just playing games"** — False; it's structured, documented, and repeatable
- **"Compliance testing is optional"** — False; required for platform certification
- **"Automation replaces all manual testing"** — False; games need extensive exploratory testing

## 🎓 Pre-Exam Checklist

- [ ] All mock exams completed with 80%+
- [ ] Glossary terms memorized
- [ ] Exam traps reviewed
- [ ] Case studies completed

## 🎓 Exam Day Advice

- Read each question fully before answering
- Watch for absolutes ("always," "never") — they are usually wrong
- For "choose N" questions, verify you selected exactly N
