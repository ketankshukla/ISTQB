## Rationales

### Q1
**B.** Supervised learning uses labeled input-output pairs. Unsupervised finds patterns. Reinforcement uses rewards. All ML requires data.

### Q2
**C.** Prioritizing by author seniority rather than actual risk is bias. Hallucination would involve inventing tests. Reasoning error would involve incorrect logic about coverage.

### Q3
**C.** Prompt chaining sequences dependent prompts. Few-shot provides examples. CoT shows reasoning. System prompting sets behavior.

### Q4
**B.** Removing/tokenizing PII is essential before public API use. Temperature doesn't mask data. Limited sharing still risks exposure. Blind trust is dangerous.

### Q5
**B.** Embeddings enable semantic similarity search. They don't compress for training, encrypt, or summarize.

### Q6
**A.** Fine-tuning updates weights; RAG doesn't. They are different approaches. RAG doesn't require retraining. Fine-tuning requires training data.

### Q7
**B.** Inconsistent format violates consistency. The scenarios may be correct, complete, and relevant — but inconsistently formatted.

### Q8
**C.** EU AI Act imposes high-risk obligations. GDPR is data protection. ISO 42001 is management systems. NIST CSF is cybersecurity.

### Q9
**B.** Missing output format causes inconsistent formatting. Role is present. Temperature and API endpoints aren't required components.

### Q10
**B.** Prompt versioning tracks changes over time. Red teaming is adversarial testing. Cost management tracks spend. Distillation creates smaller models.

### Q11
**B.** Inventing a false syllabus requirement is hallucination. Context window causes truncation. Bias involves systematic skew. Temperature affects randomness.

### Q12
**B.** Right-sizing, caching, and green hosting balance effectiveness and responsibility. Large models, no caching, and personal laptops are poor approaches.

### Q13
**A.** Assessment evaluates readiness. Pilot runs experiments. Scaling broadens. Maturity optimizes.

### Q14
**B.** Redefining roles and showing career enhancement is constructive. Devaluing skills, mandates, and replacements are poor practices.

### Q15
**B.** Smaller models and caching address latency. Large models, ensembles, and unaccelerated on-premise deployment are slow.

### Q16
**B.** Role, task, and input are present. Constraints, examples, and format are absent. The prompt is incomplete.

### Q17
**B.** Incorrect risk classification is a reasoning error. Hallucination would involve inventing features. Bias involves systematic skew. Context window causes truncation.

### Q18
**B.** Hidden instructions in uploaded content manipulating the model is indirect prompt injection. Direct is attacker-provided. Model inversion extracts training data. Supply chain attacks target dependencies.

### Q19
**B.** ISO/IEC 25059 extends ISO 25010 with AI quality characteristics. It is not EU regulation, doesn't replace ISTQB, and applies to all AI.

### Q20
**B.** Model evaluation detects output drift. Prompt versioning checks for accidental changes but doesn't explain degradation. Cost analysis and red teaming address different issues.

### Q21
**B.** Foundation models are large, general-purpose, and adaptable. They are not small, don't require immediate fine-tuning, and have limitations.

### Q22
**B.** RAG with re-ingestion handles frequent updates. Bi-weekly fine-tuning is impractical. Zero-shot lacks knowledge. Custom models are overkill.

### Q23
**B.** Governance prevents privacy breaches, quality issues, and non-compliance. Free tools, on-premise-only, and automatic provider governance are incorrect.

### Q24
**C.** Maturity involves specialization and sharing. Assessment evaluates readiness. Pilot runs first experiments. Provider selection is early.

### Q25
**B.** On-premise keeps all data in-house. Public APIs, cloud RAG, and cloud fine-tuning all risk data leaving the infrastructure.

### Q26
**A, C.** PII in defect descriptions and API retention for training are real risks. LLMs don't auto-fix defects. Grammar perfection isn't a risk.

### Q27
**B.** Involvement, training, and celebrating wins are constructive. Eliminating budgets, secrecy, and transfers are poor practices.

### Q28
**B.** CoT encourages step-by-step reasoning. Token reduction, encryption, and auto-fine-tuning are not CoT purposes.

### Q29
**B.** Hallucinations in test plans reference fake standards. Length, format inability, and no review are incorrect statements.

### Q30
**B.** Exceeding context windows causes truncation/errors. They are not unlimited. They apply to inference. They do impact prompt design.

### Q31
**A.** A/B testing compares prompt quality. Neuron count, internet speed, and implementation language are irrelevant.

### Q32
**B.** Stable format + large dataset = good fine-tuning case. RAG is for dynamic knowledge. Zero-shot wouldn't learn format. Manual writing ignores AI value.

### Q33
**B.** Privacy and security first directly addresses data protection. Iterative refinement, context, and validation are other principles.

### Q34
**B.** If the document fits, a single prompt is simplest and most effective. Chaining is unnecessary. Fine-tuning and custom models are overkill.

### Q35
**B.** Specialized, stable domain with sufficient data favors fine-tuning. Dynamic knowledge, low cost, and source citation favor RAG.

### Q36
**A.** Concrete examples directly address vagueness. Higher temperature, removing format, and fewer test cases worsen the problem.

### Q37
**C.** NIST AI RMF has Govern/Map/Measure/Manage. EU AI Act is regulation. ISO 42001 is management systems. OWASP is security risks.

### Q38
**B.** Ownership and escalation paths ensure accountability. Unlogged usage, eliminating docs, and preventing flagging create unmanaged risk.

### Q39
**B.** Misclassifying out-of-range dates as valid is a reasoning error. Hallucination would involve inventing dates. Bias involves systematic skew. Privacy breach involves exposing data.

### Q40
**B.** Augmentation and essential human judgment align with ISTQB. Elimination, unquestioning acceptance, and replacement are incorrect.
