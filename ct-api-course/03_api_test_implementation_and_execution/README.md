# Chapter 3 — API Test Implementation and Execution

## Exam Weighting

- **Estimated weight:** ~25% (~10 questions out of 40)
- **K-levels:** K2 (understand), K3 (apply)
- **Focus:** Test scripts, test data, environments, execution, defect reporting

---

## Learning Objectives

By the end of this chapter, you should be able to:

1. Implement API test scripts using various tools
2. Manage test data for API testing
3. Set up and configure API test environments
4. Execute API tests and analyze results
5. Report and track API defects effectively

---

## Key Concepts

- Test scripts for REST, SOAP, GraphQL
- Test data: static, dynamic, generated
- Environment setup: local, staging, production-like
- Test execution: manual, automated, CI/CD
- Defect reporting for APIs
