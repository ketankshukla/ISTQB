## How to Use This Course

1. **Start with `STUDY_PLAN.md`** — Pick the 4-week or accelerated schedule
2. **Read chapters in order** — Each chapter builds on the previous
3. **Study `lessons.md` first** — Read the detailed explanations and worked examples
4. **Review `exam_traps.md`** — Learn the common mistakes before you make them
5. **Practice with `practice_questions.md`** — 20 questions per chapter with full rationales
6. **Do the case studies** — Integrated scenarios that span multiple chapters
7. **Take the mock exams** — Simulate exam conditions (40 questions, 90 minutes)
8. **Track progress in `PROGRESS_TRACKER.md`** — Know when you are exam-ready

## Key Topics Covered

- **API fundamentals** — REST, SOAP, GraphQL, HTTP methods, status codes
- **API test design** — Equivalence partitioning, boundary value, state transition for APIs
- **API test implementation** — Test scripts, data management, environment setup
- **API test automation** — Frameworks, CI/CD integration, maintainability
- **API security testing** — Authentication, authorization, injection attacks
- **API performance testing** — Load, stress, soak testing for APIs
- **Contract testing** — Consumer-driven contracts, schema validation

## Tip for Maximum Exam Success

> **Do not just read. Practice.** After each chapter, answer the practice questions without looking at the answers. Then read the rationales to understand WHY the correct answer is right and WHY the distractors are wrong. This active recall is the single most effective study technique for multiple-choice exams.
