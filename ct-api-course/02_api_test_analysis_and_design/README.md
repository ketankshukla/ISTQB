# 🔬 Chapter 2 — API Test Analysis and Design

## 🎓 Exam Weighting

- **Estimated weight:** ~20% (~8 questions out of 40)
- **K-levels:** K2 (understand), K3 (apply)
- **Focus:** Test design techniques for APIs, deriving test conditions, positive/negative testing, state transition

---

## 🎯 Learning Objectives

By the end of this chapter, you should be able to:

1. Derive test conditions from API specifications
2. Apply equivalence partitioning to API parameters
3. Apply boundary value analysis to API inputs
4. Design positive and negative API test cases
5. Apply state transition testing to APIs with sessions

---

## ⭐ Key Concepts

- Test conditions from API specs (OpenAPI, WSDL)
- Equivalence partitioning for API parameters
- Boundary value analysis for numeric/string inputs
- Positive vs. negative testing
- State transition for session-based APIs
- Error guessing for APIs
