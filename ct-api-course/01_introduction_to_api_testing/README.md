# Chapter 1 — Introduction to API Testing

## Exam Weighting

- **Estimated weight:** ~10% (~4 questions out of 40)
- **K-levels:** K2 (understand)
- **Focus:** API types (REST, SOAP, GraphQL), HTTP methods, status codes, API documentation

---

## Learning Objectives

By the end of this chapter, you should be able to:

1. Explain what an API is and why APIs need testing
2. Describe different API types (REST, SOAP, GraphQL)
3. Explain HTTP methods and status codes
4. Describe API architecture patterns
5. Explain the importance of API documentation

---

## Key Concepts

- API types: REST, SOAP, GraphQL, RPC, WebSocket
- HTTP methods: GET, POST, PUT, DELETE, PATCH
- HTTP status codes: 1xx, 2xx, 3xx, 4xx, 5xx
- API architecture: monolith, microservices, SOA
- API documentation: OpenAPI, Swagger, WSDL
