# Chapter 6 — Contract Testing and Advanced Topics

## Exam Weighting

- **Estimated weight:** ~10% (~4 questions out of 40)
- **K-levels:** K2 (understand), K3 (apply)
- **Focus:** Consumer-driven contracts, schema validation, API versioning, backward compatibility

---

## Learning Objectives

By the end of this chapter, you should be able to:

1. Explain consumer-driven contract testing
2. Describe schema validation techniques
3. Explain API versioning strategies
4. Describe backward compatibility testing
5. Explain API documentation and discovery tools

---

## Key Concepts

- Consumer-driven contract testing (Pact)
- Schema validation: JSON Schema, XML Schema
- API versioning: URL, header, parameter
- Backward compatibility
- API discovery and documentation
