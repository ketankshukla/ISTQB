# ⚡ Chapter 5 — API Security and Performance Testing

## 🎓 Exam Weighting

- **Estimated weight:** ~15% (~6 questions out of 40)
- **K-levels:** K2 (understand), K3 (apply)
- **Focus:** Authentication, authorization, common vulnerabilities, performance test types

---

## 🎯 Learning Objectives

By the end of this chapter, you should be able to:

1. Explain API authentication mechanisms
2. Identify common API security vulnerabilities
3. Describe API performance test types
4. Apply security testing techniques to APIs
5. Apply performance testing techniques to APIs

---

## ⭐ Key Concepts

- Authentication: Basic, Bearer, OAuth2, API keys, JWT
- Security vulnerabilities: injection, broken auth, sensitive data exposure
- Performance tests: load, stress, soak, spike
- Rate limiting and throttling
- Security headers
