# 🤝 Chapter 5 — Regulation Acceptance Testing (RAT)

## 🎓 Exam Weighting

- **Estimated weight:** ~15% (~6 questions out of 40)
- **K-levels:** K2 (understand), K3 (apply)
- **Focus:** Compliance, standards, accessibility, data protection, audit

---

## 🎯 Learning Objectives

By the end of this chapter, you should be able to:

1. Describe Regulation Acceptance Testing (RAT) objectives
2. Identify regulatory and standard requirements affecting acceptance
3. Explain compliance testing approaches
4. Describe accessibility and data protection acceptance testing
5. Apply RAT techniques to scenarios

## ⭐ Key Concepts

- Regulation Acceptance Testing (RAT)
- Compliance requirements
- Regulatory bodies and standards
- Accessibility testing (WCAG)
- Data protection (GDPR, HIPAA, PCI-DSS)
- Audit and certification
- Industry-specific regulations
