# 📝 CT-AcT Mock Exam 2

**Instructions:** 40 questions, 60 minutes, single best answer.

---

### 🔷 Q1-Q10

1. Which is part of the CIA triad: A) Authentication B) Availability C) Authorization D) Auditing — **Availability**
2. A system encrypts data but doesn't verify tampering. Missing: A) Confidentiality B) Integrity C) Availability D) Authentication — **Integrity**
3. Verifying identity is: A) Authorization B) Authentication C) Auditing D) Accounting — **Authentication**
4. Defense in depth means: A) One strong control B) Multiple layers C) Complex design D) Secret design — **Multiple layers**
5. Least privilege: A) Admin for all B) Minimum access necessary C) Invisible security D) Expensive controls — **Minimum access necessary**
6. STRIDE Spoofing violates: A) Integrity B) Authentication C) Confidentiality D) Availability — **Authentication**
7. STRIDE Tampering violates: A) Authentication B) Integrity C) Confidentiality D) Availability — **Integrity**
8. DREAD Exploitability measures: A) Damage B) Ease of exploitation C) Users affected D) Discoverability — **Ease of exploitation**
9. Penetration test with no knowledge: A) White box B) Gray box C) Black box D) Glass box — **Black box**
10. Horizontal privilege escalation: A) Higher privileges B) Same level, different user C) Admin access D) Root access — **Same level, different user**

### 🔷 Q11-Q20

11. `<script>alert('xss')</script>` tests: A) SQL injection B) XSS C) Command injection D) Path traversal — **XSS**
12. Source code analysis without execution: A) DAST B) SAST C) IAST D) WAF — **SAST**
13. Fuzzing is: A) Manual review B) Automated random input testing C) Social engineering D) Network scanning — **Automated random input testing**
14. CVSS stands for: A) Common Verification System B) Common Vulnerability Scoring System C) Certified Scanning Standard D) Comprehensive Service — **Common Vulnerability Scoring System**
15. CVSS 9.5 means: A) Low B) Medium C) High D) Critical — **Critical**
16. False positive: A) Missed vulnerability B) Incorrectly reported vulnerability C) Correct detection D) Skipped test — **Incorrectly reported vulnerability**
17. OWASP ZAP is: A) SAST B) DAST C) SCA D) IAST — **DAST**
18. SonarQube is: A) DAST B) SAST C) Network scanner D) Fuzzer — **SAST**
19. Dependency vulnerability scanning: A) SAST B) DAST C) SCA D) IAST — **SCA**
20. Risk formula: A) Threat + Vulnerability B) Likelihood x Impact C) Control - Asset D) Attack / Defense — **Likelihood x Impact**

### 🔷 Q21-Q30

21. Attack tree OR node: A) All required B) Any child achieves goal C) Two needed D) None valid — **Any child achieves goal**
22. Risk treatment: Mitigate means: A) Ignore B) Implement controls to reduce risk C) Transfer D) Accept — **Implement controls to reduce risk**
23. A firewall blocking traffic is: A) Detective B) Preventive C) Corrective D) Compensating — **Preventive**
24. IDS alerting on suspicious activity: A) Preventive B) Detective C) Corrective D) Deterrent — **Detective**
25. Backup after ransomware: A) Preventive B) Detective C) Corrective D) Deterrent — **Corrective**
26. Warning banners: A) Preventive B) Detective C) Deterrent D) Corrective — **Deterrent**
27. "Hash passwords with bcrypt" is: A) Functional requirement B) Security requirement C) Performance D) Usability — **Security requirement**
28. Separation of duties prevents: A) Faster systems B) One person controlling critical process C) More employees D) Simpler code — **One person controlling critical process**
29. Economy of mechanism suggests: A) Maximum complexity B) Simpler = fewer vulnerabilities C) More features D) Expensive solutions — **Simpler = fewer vulnerabilities**
30. Open design principle: A) Keep design secret B) Security not dependent on secrecy C) Publish passwords D) Hide vulnerabilities — **Security not dependent on secrecy**

### 🔷 Q31-Q40

31. Complete mediation: A) Check only at login B) Every access checked C) Never challenge D) Optional checks — **Every access checked**

33. Psychological acceptability: A) Security at any cost B) Security shouldn't impede usability C) Only for experts D) Ignore users — **Security shouldn't impede usability**
34. IAST is: A) Source analysis B) Outside-in testing C) Instrumented running code D) Network scanning — **Instrumented running code**
35. Nmap is for: A) Source analysis B) Network discovery and port scanning C) Database encryption D) Web fuzzing — **Network discovery and port scanning**
36. GitLeaks scans: A) SQL injection B) Git history for secrets C) Network ports D) Java bytecode — **Git history for secrets**
37. Security gate in CI/CD: A) Always pass B) Block on critical failures C) Only manual D) Only code style — **Block on critical failures**
38. Container scanning detects: A) Performance issues B) Known CVEs in images C) Container size D) Labels — **Known CVEs in images**
39. Passive DAST scanning: A) Sends payloads B) Analyzes traffic without payloads C) Only HTTPS D) Needs source code — **Analyzes traffic without payloads**
40. Burp Suite: A) Network scanner B) Web app security testing C) Compiler D) Database optimizer — **Web app security testing**
