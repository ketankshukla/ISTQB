# CT-AcT Mock Exam 1

**Instructions:** 40 questions, 60 minutes, single best answer.

---

### Q1-Q10

1. The PRIMARY purpose of acceptance testing is to: A) Find coding bugs B) Validate system meets business needs and acceptance criteria C) Optimize performance D) Review code — **Validate system meets business needs and acceptance criteria**
2. User Acceptance Testing (UAT) is performed by: A) Developers B) Users or business representatives C) Auditors D) Only testers — **Users or business representatives**
3. Operational Acceptance Testing (OAT) focuses on: A) User interface design B) Operational aspects like backup, recovery, maintenance C) Code review D) Unit testing — **Operational aspects like backup, recovery, maintenance**
4. Beta testing is: A) Internal developer testing B) External user testing at their own locations C) Only QA team testing D) No user involvement — **External user testing at their own locations**
5. Good acceptance criteria should be: A) Vague B) Testable, clear, business-focused, specific, realistic C) Technical only D) Written only by developers — **Testable, clear, business-focused, specific, realistic**
6. Given-When-Then format is used for: A) Code comments B) BDD acceptance criteria C) Database schemas D) Network diagrams — **BDD acceptance criteria**
7. The acceptance decision should be made by: A) Development team B) Business representatives or authorized stakeholders C) Testers only D) Operations team — **Business representatives or authorized stakeholders**
8. Alpha testing is performed: A) By users at their own locations B) By developers or internal team at developer's site C) After beta testing D) Only after production release — **By developers or internal team at developer's site**
9. The PRIMARY difference between system testing and acceptance testing: A) System is shorter B) System tests technical specs; acceptance validates business needs C) Acceptance is first D) System has no documentation — **System tests technical specs; acceptance validates business needs**
10. Contract Acceptance Testing (CAT) validates: A) User satisfaction only B) Contractual requirements between supplier and customer C) Only performance D) Only security — **Contractual requirements between supplier and customer**

### Q11-Q20

11. In the V-model, acceptance testing corresponds to: A) Coding B) Requirements specification C) Detailed design D) Architecture design — **Requirements specification**
12. In agile, acceptance testing is: A) Only at project end B) Integrated continuously throughout each sprint C) Not needed D) Only by Scrum Master — **Integrated continuously throughout each sprint**
13. Entry criteria define: A) When testing is complete B) Conditions that must be met before testing can begin C) Test cases D) Defect severity — **Conditions that must be met before testing can begin**
14. Exit criteria define: A) When to start testing B) Conditions for testing to be considered complete C) Only pass rates D) Only defect counts — **Conditions for testing to be considered complete**
15. Traceability matrix maps: A) Only defects to code B) Requirements to tests and defects C) Only developers to tasks D) Only budget to schedule — **Requirements to tests and defects**
16. ATDD means: A) Acceptance tests after development B) Acceptance tests defined before development begins C) Only manual testing D) No testing needed — **Acceptance tests defined before development begins**
17. The Definition of Done includes: A) Only code written B) Acceptance criteria met and Product Owner review complete C) Only unit tests D) Only deployment — **Acceptance criteria met and Product Owner review complete**
18. Sprint review is: A) Formal acceptance testing B) Informal demo and feedback session C) Only for developers D) The same as retrospective — **Informal demo and feedback session**
19. Shift-left in acceptance testing means: A) Testing only at the end B) Starting acceptance activities earlier in the lifecycle C) Moving testing to production D) Eliminating testing — **Starting acceptance activities earlier in the lifecycle**
20. Forward traceability means: A) Defects to requirements B) Requirements to tests to results C) Code to design D) Tests to code — **Requirements to tests to results**

### Q21-Q30

21. Business Acceptance Testing (BAT) verifies: A) Only code quality B) System meets business needs and requirements C) Only security D) Only performance — **System meets business needs and requirements**
22. A user story format is: A) "As a [role], I want [feature] so that [benefit]" B) Only technical specifications C) Only test cases D) Only database schemas — **"As a [role], I want [feature] so that [benefit]"**
23. BDD's "Three Amigos" are: A) Three test tools B) Business, developer, and tester C) Three project phases D) Three types of testing — **Business, developer, and tester**
24. A pilot deployment: A) Replaces all testing B) Validates system in production with limited group before full rollout C) Trains all users at once D) Skips acceptance testing — **Validates system in production with limited group before full rollout**
25. Sign-off can occur with: A) Zero defects only B) Known acceptable risks and remediation plans C) Only cosmetic issues D) Only after 100% automation — **Known acceptable risks and remediation plans**
26. UAT test data should be: A) Same as unit test data B) Realistic and production-like C) Completely random D) Only invalid data — **Realistic and production-like**
27. A critical UAT defect is one that: A) Has a typo B) Prevents a business process from completing with no workaround C) Is cosmetic D) Only affects one user — **Prevents a business process from completing with no workaround**
28. ATDD acceptance tests are written: A) After coding B) Before development begins C) During system testing D) After UAT — **Before development begins**
29. Canary release is: A) Full deployment B) Deploy to small percentage of users, monitor, expand C) Only system test D) Only for web apps — **Deploy to small percentage of users, monitor, expand**
30. A phased rollout means: A) All users at once B) One department or location at a time C) No testing D) Only internal users — **One department or location at a time**

### Q31-Q40

31. Contractual acceptance criteria come from: A) Developer preferences B) Contract, SOW, agreed specifications C) Only UI design D) Only tester opinions — **Contract, SOW, agreed specifications**
32. An SLA typically: A) Is only for acceptance B) Is an ongoing service commitment C) Only applies to development D) Is not measurable — **Is an ongoing service commitment**
33. Liquidated damages are: A) A bonus B) Pre-agreed compensation for contract breaches C) A type of insurance D) A test technique — **Pre-agreed compensation for contract breaches**
34. Outsourced project acceptance should be performed by: A) Only the vendor B) The customer to independently verify C) Nobody D) Only auditors — **The customer to independently verify**
35. Regulation Acceptance Testing (RAT) verifies: A) Only functionality B) Compliance with laws, regulations, standards C) Only performance D) Only usability — **Compliance with laws, regulations, standards**
36. WCAG 2.1 AA is: A) Lowest level B) Intermediate level commonly required C) Highest level D) Not real — **Intermediate level commonly required**
37. GDPR "right to erasure" means: A) Users erase hard drives B) Users request deletion of personal data C) Companies erase complaints D) Testers erase data — **Users request deletion of personal data**
38. PCI-DSS applies to: A) All software B) Systems handling payment card data C) Only web apps D) Only mobile apps — **Systems handling payment card data**
39. Accessibility testing includes: A) Only automated scans B) Automated, manual, and user testing C) Only performance D) Only security — **Automated, manual, and user testing**
40. Cucumber is a: A) Unit test framework B) BDD tool for acceptance tests C) Performance tool D) Database tool — **BDD tool for acceptance tests**
