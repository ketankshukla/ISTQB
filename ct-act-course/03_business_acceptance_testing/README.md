# Chapter 3 — Business Acceptance Testing (BAT)

## Exam Weighting

- **Estimated weight:** ~20% (~8 questions out of 40)
- **K-levels:** K2 (understand), K3 (apply)
- **Focus:** UAT, acceptance criteria, user stories, BDD, sign-off

---

## Learning Objectives

By the end of this chapter, you should be able to:

1. Plan and design Business Acceptance Testing (BAT)
2. Write effective acceptance criteria for user stories
3. Describe UAT preparation, execution, and evaluation
4. Explain ATDD and BDD approaches
5. Describe the sign-off process and pilot deployments
6. Apply BAT techniques to scenarios

---

## Key Concepts

- Business Acceptance Testing (BAT)
- User Acceptance Testing (UAT)
- Acceptance criteria formats (Given-When-Then, checklists, rules)
- User stories and story mapping
- ATDD and BDD
- UAT environment and data
- UAT execution and defect management
- Sign-off and pilot
