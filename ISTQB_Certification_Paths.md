# 🎯 ISTQB Certification Portfolio & Recommended Learning Paths

## 🌟 Overview

This document maps the complete ISTQB certification landscape and provides recommended paths for testers at different career stages. Use it to plan your certification journey.

---

## 🪜 Foundation Level (Entry Point — Mandatory Prerequisite)

| Certification | Focus | Who It's For | Prerequisites |
|-------------|-------|-------------|---------------|
| **CTFL v4.0** | Core testing fundamentals: testing principles, test levels, test types, test techniques, test management, tool support | Everyone entering software testing | None |
| **CTFL-AT** | Agile testing principles: agile values, agile testing practices, continuous testing, BDD/ATDD, exploratory testing | Teams using Scrum, Kanban, SAFe, or other agile frameworks | None |
| **CTFL-MBT** | Model-based testing: state transition, decision tables, combinatorial testing, model coverage | Teams using formal models, state-based testing, BDD with models | None |
| **CTFL-GT** | Gambling industry testing: RNG, game math, compliance, player protection, responsible gaming | iGaming, lottery, casino software testing | None |
| **CTFL-PT** | Foundation performance testing: load, stress, spike, endurance, scalability, volume, monitoring, bottleneck analysis | Testers new to performance testing | None |
| **CTFL-SEC** | Foundation security testing: CIA triad, threats, vulnerabilities, SAST, DAST, penetration testing basics, OWASP | Testers new to security testing | None |
| **CTFL-AuT** | Foundation automotive testing: ECUs, vehicle networks, ISO 26262 basics, ASIL, ASPICE, MIL/SIL/PIL/HIL | Testers new to automotive software testing | None |

**Rule:** You must hold ONE Foundation Level certification before attempting any Advanced or Specialist certification.

## 🪜 Advanced Level (Requires Foundation Level)

| Certification | Focus | Target Role |
|-------------|-------|-------------|
| **CTAL-TM** | Test management: strategy, planning, estimation, risk-based testing, defect management, team management, test process improvement | Test Managers, Test Leads, QA Directors |
| **CTAL-TA** | Test analysis & design: advanced test techniques, quality characteristics, reviews, test tools, technical testing | Senior Test Analysts, Test Engineers, QA Specialists |
| **CTAL-TTA** | Technical test analyst: white-box testing, static/dynamic analysis, non-functional testing, test tools, CI/CD integration | SDETs, Technical Testers, Automation Leads |
| **CTAL-AT** | Agile technical testing: TDD, CI/CD testing, DevOps, continuous delivery, test automation in agile | Agile Testers, DevOps Engineers, Agile Coaches |

**Note:** You can take Advanced certifications in any order after Foundation. Each is standalone — you do not need CTAL-TM before CTAL-TA.

## ⭐ Recommended Beginner Paths

### 🔷 Path A: The Classic Test Analyst
**CTFL v4.0 → CTAL-TA → CT-UT or CT-AcT**

Best for: Manual testers, business testers, QA analysts who want to deepen their test design and analysis skills.
- CTFL gives you the foundation
- CTAL-TA teaches advanced techniques (equivalence partitioning, boundary value, decision tables, state transition)
- Add CT-UT for accessibility/UX specialization or CT-AcT for agile acceptance testing

**Time estimate:** 3-4 months per certification

---

### 🏃 Path B: The Agile Tester
**CTFL v4.0 → CTFL-AT → CTAL-AT → CT-API**

Best for: Teams in agile environments who need testing embedded in sprint delivery.
- CTFL gives you the foundation
- CTFL-AT teaches agile-specific testing practices
- CTAL-AT adds technical depth (TDD, CI/CD, DevOps testing)
- CT-API rounds out modern agile stack (microservices, contract testing)

**Time estimate:** 2-3 months per certification

---

### ⚙️ Path C: The Automation Engineer
**CTFL v4.0 → CT-TAE → CTAL-TTA → CT-API**

Best for: SDETs, automation engineers, CI/CD pipeline owners.
- CTFL gives you the foundation
- CT-TAE teaches test automation architecture (framework design, maintainability)
- CTAL-TTA adds white-box testing, static analysis, non-functional testing
- CT-API covers service layer automation

**Time estimate:** 3-4 months per certification

---

### ⚡ Path D: The Security/Performance Specialist
**CTFL v4.0 → CT-SEC or CT-PT → CTAL-TTA**

Best for: Testers in regulated industries, fintech, healthcare, gaming, or high-load systems.
- CTFL gives you the foundation
- CT-SEC or CT-PT provides deep specialization
- CTAL-TTA adds technical analysis depth for advanced security/performance scenarios

**Time estimate:** 3-4 months per certification

---

### 🤖 Path E: The AI-First Modern Tester
**CTFL v4.0 → CT-GenAI → CT-TAE → CT-API**

Best for: Testers who want to stay current with modern tooling and AI-augmented workflows.
- CTFL gives you the foundation
- CT-GenAI teaches LLM-powered testing (prompt engineering, risk management, adoption)
- CT-TAE provides the automation architecture to integrate AI tools into pipelines
- CT-API covers the service layer that most AI tools interact with

**Time estimate:** 3 months per certification

---

### 🗂️ Path F: The Test Manager
**CTFL v4.0 → CTAL-TM → CTAL-TA**

Best for: Test leads and managers who need to plan, estimate, and steer testing strategy.
- CTFL gives you the foundation
- CTAL-TM teaches test management, strategy, estimation, risk, and improvement
- CTAL-TA adds technical depth so you can review and guide test design decisions

**Time estimate:** 3-4 months per certification

---

## ⭐ Certification Sequence Rules

1. **Foundation is mandatory first.** No exceptions.
2. **Advanced can be taken in any order** after Foundation. CTAL-TM does not require CTAL-TA.
3. **Specialist can be taken in any order** after Foundation. Some have recommended prerequisites (e.g., CT-MBT recommends CTFL-MBT).
4. **No expiration.** ISTQB certifications do not expire. Once you pass, you're certified for life.
5. **Syllabus versions matter.** Always study the current syllabus version (v4.0 as of 2024 for most).
